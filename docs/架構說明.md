# 架構說明

本文對齊目前程式碼結構，說明 SyncTranslate 的主要 runtime、ASR/翻譯/TTS 流程，以及最近 ASR queue 與字幕 final 處理的實作位置。

## 入口與啟動

`main.py` 是 GUI/CLI 入口。啟動流程大致如下：

1. `app/bootstrap/external_runtime.py` 設定外部 runtime 路徑。
2. `app/infra/config/settings_store.py` 載入 `config.yaml`，必要時套用 migration。
3. `app/bootstrap/app_factory.py` 建立 pipeline bundle。
4. `app/ui/main_window.py` 建立 PySide6 UI 並掛上 session/control callbacks。

## 音訊與字幕管線

```text
AudioCapture
  -> AudioInputManager
  -> AudioRouter
  -> ASRManagerV2
  -> SourceRuntimeV2
  -> EndpointingRuntime
  -> FasterWhisper backend pair
  -> TranscriptPostProcessor
  -> TranscriptBuffer
  -> TranslationDispatcher / TranslatorManager
  -> TTSManager / UI
```

核心協調在 `app/application/audio_router.py`：

- 接收 local/remote 音訊 chunk。
- 依 runtime state 決定是否送進 ASR。
- 收到 ASR partial/final 後寫入原文字幕 buffer。
- final 文字會先經 `TranscriptBuffer.preview_final_text()` 去除與上一句 final 的重疊，再送翻譯。
- 翻譯結果寫入 translated channel，必要時送 TTS。

字幕儲存在 `app/application/transcript_service.py`。相鄰 final 的重疊前綴會在 `_trim_overlap_with_previous_final_locked()` 處理，避免上一句句尾與下一句句頭重複。

## local / remote 語意

- `remote`：遠端/會議音訊，UI 顯示為遠端原文與遠端翻譯。
- `local`：本地麥克風或本地音訊，UI 顯示為本地原文與本地翻譯。

TTS/passthrough 的輸出方向由 `AudioRouter._channels_of()` 決定：

- `remote` ASR 的翻譯可送到 `local` 輸出。
- `local` ASR 的翻譯可送到 `remote` 輸出。

## ASR v2

ASR 入口是 `app/infra/asr/manager_v2.py`，每個 source 建立一個 `SourceRuntimeV2`。

主要模組：

- `backend_resolution.py`：依語言解析 disabled/chinese/non_chinese/auto。
- `profile_selection.py`：依語言 family 選擇 ASR profile slot。
- `language_profiles.py`：覆寫各語言的 beam、VAD、streaming、frontend 參數。
- `backend_v2.py`：建立 partial/final faster-whisper backend。
- `faster_whisper_adapter.py`：呼叫 faster-whisper/CTranslate2，並處理模型快取、CUDA/CPU fallback、prompt/context。新增 `SegmentMetadata(text, start, end, avg_logprob, no_speech_prob, compression_ratio)` dataclass 與擴充後的 `TranscribeResult`（含 `segments`、`avg_logprob`、`max_no_speech_prob`、`max_compression_ratio`）；`_collect_segment_metadata()` 跨 segment 聚合信心指標，avg_logprob 以 duration 加權平均，no_speech_prob 與 compression_ratio 取各 segment 最大值。
- `endpointing_v2.py`：Silero VAD/RMS endpoint runtime。
- `endpoint_profiles.py`：`meeting_room`、`turn_taking`、`max_accuracy` 等 endpoint preset。
- `worker_v2.py`：queue、音訊前端、partial/final decode、背壓、重啟 segment、final fallback。新增 `_should_suppress_by_no_speech(result, text)` 在 no_speech_prob > 0.75 且 CJK 字元數為 0 且文字過短時壓制輸出；新增 `_has_compression_loop_risk(result)` 在 compression_ratio > 2.6 時標記潛在 loop 風險。
- `streaming_policy.py`：根據 VAD signal、backlog、segment 長度與 degradation 狀態決定 emit partial/final；queue 壓力高時降低或暫停 partial。新增 **final priority 狀態機**：`queue_ratio >= 0.45`、final 延遲超過門檻或 degraded+drops 時進入，完全壓制所有 partial，並將 finalization threshold 減半，讓 worker 更積極輸出 final；queue ratio 回落到 0.15 且持續 8s 後才退出。`StreamingDecision` 新增欄位 `final_priority_active` 與 `final_priority_reason`。
- `auto_language_state.py`：`AutoLanguageState` dataclass，追蹤 `auto` 模式下每個 source 的動態語言偵測狀態（`chinese_streak`、`non_chinese_streak`、`stable_family`、`last_switch_ms` 等）。純函式 `observe_final_language()` 依 CJK/Latin 字元比例將 utterance 分類為 chinese/non_chinese/mixed，連續 3 次相同 family 且超過冷卻期（25s）後回傳新 effective_language，否則回傳 `None`；太短的 utterance 略過（中文少於 4 CJK 字、非中文少於 3 詞）。
- `frontend_v2.py` / `enhancement_v2.py`：mono collapse、DC removal、high-pass、AGC、頻譜增強。
- `transcript_validator_v2.py` / `_hallucination_filter.py`：過濾過密文字、loop、字幕 overlay、尾端幻覺。

`manager_v2.py` 維護每個 source 的 `AutoLanguageState`；`_observe_detected_language()` 在每個 final event 後更新狀態；`_maybe_rebuild_runtime_for_language_switch()` 在 effective_language 變更時觸發 runtime 重建，並設有 30s force-rebuild 安全閥防止短時間內過度重建。`stats()` 新增 `auto_language` 字典（含 `requested`/`effective`/`detected`/`family`/`pending_rebuild`）及 `final_priority_active`/`final_priority_reason` 欄位。

## 顯示層後處理

`app/application/display_punctuation.py` 提供 `apply_display_punctuation()` 輕量標點補全引擎，僅作用於 UI 渲染層，不影響翻譯管線。規則刻意保守以避免誤判：

- CJK 字元數少於 4 的片段直接略過（過短填充詞）。
- 結尾含問詞（嗎、吧、呢、什麼、怎等）時補全全形 `？`。
- CJK 字元數達到 8 以上的長段補全 `。`。
- 已有句末標點（。？！…等）時不疊加。
- 僅處理 final（`is_final=True`）；partial 原樣透傳。

`app/ui/main_window.py` 的 `refresh_live_caption` 在 `asr_display_punctuation_enabled = True` 時對 final 行呼叫 `apply_display_punctuation()`，作用語言由 `asr_display_punctuation_languages` 控制，模式由 `asr_display_punctuation_mode` 控制（目前 `lightweight`）。

## 語言與模型路由

目前路由規則：

- `zh`、`zh-TW`、`zh-CN`、`cmn`、`yue` 屬於 Chinese family。
- Chinese family 使用 `asr_channels.local` profile，預設模型是 `large-v3-turbo`；`.\runtimes\models\belle-zh-ct2` 保留為設定 UI 可選備用。
- 非中文與 `auto` 使用 `asr_channels.remote` profile，預設模型是 `large-v3-turbo`。
- `none` 會停用該 channel ASR。

中文 language profile 的重點：

- `condition_on_previous_text = False`
- `final_condition_on_previous_text = False`
- `temperature_fallback = "0.0"`
- `no_speech_threshold = 0.25`
- Silero VAD threshold `0.45`
- endpoint profile `meeting_room`
- frontend 保留基本處理，但 `enhancement_enabled = False`

停用中文頻譜增強的原因是 benchmark 顯示中文 ASR 對降噪/音樂抑制後的頻譜失真較敏感，容易產生短句誤識別或重複音節；保留 AGC/high-pass 則仍有助於 VAD 與音量穩定。

## ASR queue 與背壓

queue 設定在 `runtime.asr_queue_maxsize_local` / `runtime.asr_queue_maxsize_remote`，目前建議值為 `256`。

`SourceRuntimeV2` 的背壓策略：

- `submit_chunk()` 在 queue 接近高水位時會進入 congested 狀態、暫停 partial；queue 滿時會丟最舊 chunk 並記錄 `dropped_chunks`。
- `_drain_limit_for_backlog()` 會依 queue 壓力一次合併更多 chunk，降低 Python/前端/VAD 迴圈成本。
- `_pressure_force_final_audio_ms()` 避免 queue 壓力大時太早 forced final，因為 final decode 是昂貴操作。
- `StreamingPolicy` 在 congested/degraded 狀態會壓低 partial 頻率或 suppress partial，讓 worker 優先保住 final 與音訊消化速度。
- **Final priority 模式**（`streaming_policy.py`）：queue_ratio >= 0.45、final 延遲超過 1800ms 或 degraded+drops 時自動進入，完全壓制 partial 並將 finalization threshold 減半；queue ratio 回落到 0.15 後持續 8s 才退出正常模式。

如果 log 出現：

```text
ASR queue overflow (local): dropped ... chunks total
```

代表 ASR worker 已丟音訊，會造成漏句與翻譯錯誤。偶發 `dropped 1` 可觀察；若持續遞增到 `11、21、31...`，應降低負載或調整 queue/profile。

## 翻譯

翻譯核心在 `app/infra/translation/engine.py`，主要使用：

- `inprocess_adapter.py`：`llama_cpp` in-process GGUF runtime。
- `_prompt_builder.py`：建立 JSON-only 翻譯 prompt。
- `_stream_parser.py`：解析 LLM stream。
- `stitcher.py`：partial/final context 與去重。
- `translation_dispatcher.py`：async queue，避免 ASR thread 被 LLM 阻塞。

預設模型路徑：

```text
runtimes/models/llm/hy-mt1.5-7b.gguf
```

## TTS 與音訊輸出

TTS runtime 在 `app/infra/tts/playback_queue.py`。`TTSManager` 負責：

- channel queue。
- Edge TTS 文字轉語音。
- output mode：`tts`、`subtitle_only`、`passthrough`。
- 依 utterance/revision 取消舊任務。
- passthrough 音訊轉送。

音訊 I/O 在 `app/infra/audio/`：

- `capture.py`：sounddevice input stream。
- `routing.py`：local/remote capture 管理。
- `playback.py`：播放與 passthrough。
- `device_registry.py`：裝置列舉與 selector。
- `device_volume_controller.py`：Windows endpoint volume。

## Benchmark 與回歸測試工具

ASR benchmark 工具位於 `tools/asr_benchmark/`：

- `run_multi_benchmark.py`：多樣本 benchmark（語言/profile 矩陣）。
- `run_zh_preset_matrix.py`：中文 preset × model 矩陣。
- `streaming_sim.py`：單檔 streaming simulation。
- `prepare_local_benchmark_corpus.py`：建立 curated stable baseline corpus。它保留雙模式穩定通過的本機 benchmark 段落，並補入已實測穩定的 FLEURS/ASCEND 樣本。
- `run_regression_corpus.py`：manifest 驅動的 ASR regression runner。優先讀取 `downloads/asr_regression_local/manifest.yaml`，也可讀取公開資料集 fallback 的 `downloads/asr_regression/manifest.yaml`；支援 `--models`、`--modes`、`--speed`、`--quick`、`--skip-missing` 旗標，輸出 JSON summary 與 Markdown report。

## ASR v3 final rescue

ASR v3 保留 v2 pipeline 的快速 partial 路徑，只在 final 階段做低信心救援。

- `app/infra/asr/confidence_gate.py`：用 `avg_logprob`、`max_no_speech_prob`、`max_compression_ratio`、短文字與音訊長度判斷是否需要 rescue，並負責候選結果評分。
- `app/infra/asr/model_router.py`：中文 final 的 fallback model 路由，在 belle-zh-ct2 與 large-v3-turbo 之間選擇替代模型。
- `app/infra/asr/faster_whisper_adapter.py`：保留 faster-whisper segment metadata，並提供 `transcribe_final_rescue_result()`。
- `app/infra/asr/backend_v2.py`：將 metadata 帶入 `BackendTranscript`，並提供 `transcribe_final_rescue()`。
- `app/infra/asr/worker_v2.py`：在 `_emit_final()` 內執行 confidence gate、同模型 rescue、中文 fallback 與 candidate selection。
- `app/infra/asr/manager_v2.py`：依 `runtime.asr_accuracy_mode` 建立 `FinalRescuePolicy`，並為中文通道建立 fallback backend。

```text
partial decode -> UI 即時顯示
final decode -> confidence gate
  -> 高信心：直接採用
  -> 低信心：同模型 rescue
  -> 中文且允許 fallback：替代模型 rescue
  -> candidate selection -> final event
```

診斷資料會輸出 `fallback_model`、`final_rescue_enabled`、`final_rescue_count`、`final_fallback_count` 與 `last_final_rescue_reason`。

## ASR regression corpus

ASR v3 的量測工具位於 `tools/asr_benchmark/`：

- `prepare_local_benchmark_corpus.py`：建立 35 筆 curated stable regression corpus，包含 20 筆中文、10 筆英文、5 筆中英混合；reference quality 包含 `subtitle_aligned` 與 `dataset_transcript`。高難度短影音、故事尾段與快語速混語素材應放在 stress 測試，不放進 baseline gate。
- `prepare_regression_corpus.py`：從公開資料集建立 fallback corpus，適合補足外部語料壓力測試。
- `run_regression_corpus.py`：讀取 manifest，將 `meeting` / `dialogue` 映射到實際 endpoint profile，連續執行 streaming simulation，輸出 JSON/Markdown baseline。
- `streaming_sim.py`：實際模擬 chunk feeding、endpointing、partial/final decode、ASR v3 rescue/fallback 與指標計算。

corpus 音檔與 reference 位於 `downloads/asr_regression_local/` 或 `downloads/asr_regression/`，不提交 Git；可提交的是工具、manifest 範本、測試與文件。baseline 報告預設輸出至 `downloads/benchmark_results/asr_regression*`。
