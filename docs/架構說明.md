# 架構說明

本文描述目前實際執行中的 SyncTranslate 架構。已移除的舊 ASR worker、UI controller prototype、重複 metrics/dispatcher shim 不再列入。

## 啟動流程

`main.py` 是唯一入口。它先配置外部 AI runtime 與 crash log，再交給 `app/bootstrap/app_factory.py` 建立 CLI 或 GUI。

主要步驟：

1. `configure_external_ai_runtime()` 將 `runtimes/shared`、`runtimes/faster_whisper` 與 DLL 路徑加入 runtime。
2. `load_config()` 載入 `config.yaml` 或 fallback 到 `config.example.yaml`。
3. `build_pipeline_bundle()` 建立音訊、ASR、翻譯、TTS、router 與 session controller。
4. `MainWindow` 建立 PySide6 UI，並把設定頁、字幕頁、診斷頁接到同一份 `AppConfig`。

## 即時管線

```text
AudioCapture
  -> AudioInputManager
  -> AudioRouter
  -> ASRManagerV2
  -> SourceRuntimeV2
  -> EndpointingRuntime
  -> FasterWhisper backend pair
  -> TranscriptPostProcessor
  -> TranscriptStore
  -> TranslationDispatcher
  -> TranslatorManager
  -> TTSManager / UI
```

`AudioRouter` 是核心協調者。它接收 local/remote 音訊 chunk，送入 ASR，處理 partial/final 字幕，決定是否翻譯，並依通道輸出模式送入 TTS 或旁路播放。

## 通道模型

系統固定有兩個來源：

- `remote`：會議或系統音訊。
- `local`：本機麥克風。

TTS 輸出採交叉通道：

- `remote` 來源翻譯後輸出到 `local` 播放端。
- `local` 來源翻譯後輸出到 `remote` 播放端。

這讓使用者聽到對方語音的翻譯，對方聽到使用者語音的翻譯。

## ASR

ASR 入口是 `app/infra/asr/manager_v2.py`。它依來源與語言建立 `SourceRuntimeV2`，並選擇有效 profile。

核心模組：

- `backend_resolution.py`：將語言解析成 disabled/chinese/non_chinese/auto。
- `language_profiles.py`：依語言調整 beam、VAD、endpoint、frontend 與 streaming 參數。
- `backend_v2.py`：組合 partial/final faster-whisper backend。
- `faster_whisper_adapter.py`：載入 faster-whisper/CTranslate2，處理模型快取與 fallback。
- `endpointing_v2.py`：VAD 與 endpoint decision。
- `worker_v2.py`：串流 worker，負責 queue、partial/final decode、降級策略、fallback。
- `streaming_policy.py`：根據 backlog、延遲、語音長度決定 partial/final 時機。
- `frontend_v2.py` / `enhancement_v2.py`：音訊前處理、降噪、音樂抑制、AGC。
- `_hallucination_filter.py` / `transcript_validator_v2.py`：過濾字幕幻覺與格式污染。

## 翻譯

翻譯入口是 `app/infra/translation/engine.py`。

主要元件：

- `provider.py`：provider protocol 與本地 llama provider。
- `inprocess_adapter.py`：載入 `llama_cpp` 與 GGUF 模型。
- `_prompt_builder.py`：建立 JSON-only 翻譯/修正 prompt。
- `_stream_parser.py`：解析並清理 LLM 輸出。
- `stitcher.py`：partial gating、context、快取與結果驗證。
- `translation_dispatcher.py`：背景 queue，避免 ASR thread 被 LLM 阻塞。

## TTS 與音訊

TTS 入口是 `app/infra/tts/playback_queue.py` 的 `TTSManager`。它負責：

- 每個 channel 的 TTS queue。
- Edge TTS 合成。
- 長句切段。
- 舊 revision 任務取消。
- passthrough queue。
- 播放端 selection。

音訊層在 `app/infra/audio/`：

- `capture.py`：sounddevice input stream。
- `routing.py`：local/remote capture 管理。
- `playback.py`：sounddevice/soundcard 播放與 passthrough。
- `device_registry.py`：裝置列舉與 selector。
- `_device_selector.py`：輸出裝置排序。
- `device_volume_controller.py`：Windows endpoint volume 控制。

## UI

UI 使用 PySide6：

- `main_window.py`：主視窗、session start/stop、healthcheck、設定套用、診斷匯出。
- `pages/live_caption_page.py`：即時字幕四面板。
- `pages/audio_routing_page.py`：音訊路由。
- `pages/local_ai_page.py`：ASR/LLM/TTS/runtime 設定。
- `pages/diagnostics_page.py`：健康狀態與 runtime 資訊。
- `pages/settings_page.py`：設定頁容器。

## 設定

設定層在 `app/infra/config/`：

- `schema.py` 定義 `AppConfig` 與各區塊 dataclass。
- `settings_store.py` 負責載入、儲存與 fallback。
- `_schema_parser.py` 將 YAML dict 轉成 dataclass。
- `_config_migration.py` 遷移舊設定。
- `_config_serialization.py` 做內外部欄位名稱轉換。
- `glossary_loader.py` 載入詞彙表。

## 診斷與匯出

- `app/local_ai/healthcheck.py`：ASR/LLM/TTS 本地健康檢查。
- `app/local_ai/healthcheck_worker.py`：隔離 subprocess healthcheck。
- `app/application/healthcheck_service.py`：GUI healthcheck service。
- `app/application/diagnostics_export.py`：診斷文字匯出。
- `app/application/export_service.py`：session report 匯出。

## 清理後的邊界

目前不再保留以下類型檔案：

- 舊 ASR compatibility worker。
- 未接入 MainWindow 的 UI controller prototype。
- 與 `PipelineLatencyTracker` / `TTSManager` 重複的 metrics 或 TTS dispatcher shim。
- 只測試已移除 prototype 的測試檔。
- 一次性 PR summary 與歷史流水帳文件。
