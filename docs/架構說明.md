# SyncTranslate v2 架構說明

## 核心分層

```text
UI / CLI
  -> AppConfig + migration
  -> DependencyContainer / AppFactory
  -> SessionService
  -> AudioRouter
  -> ASR / Translation / TTS / Audio sinks
```

v2 的核心不是舊 `direction.mode`，而是 `runtime.session_mode`：

- `meeting`: 單一 monitor source，跑 ASR、translation、transcript，不建立 virtual bridge/source/sink。
- `dialogue`: local source + remote virtual source，依每方向 TTS voice 決定 translation route 或 direct passthrough。

## Meeting Mode Pipeline

```text
system input or WASAPI output loopback
  -> AudioFrame(float32, source metadata)
  -> ASR adapter converts to 16 kHz mono float32
  -> faster-whisper/CTranslate2
  -> translation
  -> transcript/caption buffers
```

meeting mode 不啟動 SyncTranslate Virtual Speaker、Virtual Microphone、bridge、driver readiness check，也不做 direct passthrough。

## Dialogue Mode Pipeline

```text
remote virtual speaker source -> ASR -> translation -> TTS -> local speaker
local microphone source       -> ASR -> translation -> TTS -> virtual microphone
```

如果某方向 TTS voice 是 `none`：

```text
source AudioFrame -> direct passthrough -> output boundary
```

direct passthrough 跳過 ASR、LLM、transcript、TTS queue、gate、warmup、batch 與 synthesis backoff。

## Audio Format Contract

- App internal: float32 `AudioFrame`。
- ASR input: 16 kHz mono float32。
- Virtual audio protocol v2 / bridge / driver boundary: 48 kHz PCM16 interleaved stereo。

`app/infra/audio/bridge_protocol.py` 提供 protocol v2 packet encode/decode。Python bridge/client 與 driver client 的正式 virtual mic path 必須保持 48 kHz PCM16 stereo boundary，不可先 downmix 成 mono。

## Runtime Readiness

first-run readiness 與 validation tools 會輸出：

- `meeting_ready`
- `dialogue_ready`
- `asr_model_ready`
- `llm_model_ready`
- `bridge_ready`
- `driver_ready`
- `driver_format_status`
- `suggested_next_action`

meeting mode 中 driver/bridge missing 不算 failure。dialogue mode 中 driver/bridge/format missing 會 WARN/FAIL，並提示安裝或驗證方式。

## Driver / Bridge

Dialogue mode 的 driver contract：

- endpoint boundary: `48000Hz / PCM16 / 2ch`
- protocol version: `2`
- packet layout: interleaved stereo PCM16
- ring buffer counters: frames, not mono samples

開發測試 driver 使用 test-signed package。正式發佈需要 production driver signing / WHQL 或等價 Microsoft 認可簽章流程。

## Diagnostics

Diagnostics export 與 UI summary 應顯示 v2 runtime 狀態：

- `session_mode`
- `meeting_audio_source`
- `capture_kind`
- `capture_sample_rate`
- `capture_channels`
- `asr_input_sample_rate`
- `asr_input_channels`
- `direct_passthrough_active_local`
- `direct_passthrough_active_remote`
- `bridge_required`
- `bridge_connected`
- `driver_available`
- `last_route_policy_reason`

舊 `direction.mode` 只能以 `legacy_direction_mode` 標示，不能作為主要產品狀態。
