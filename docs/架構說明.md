# 架構說明

> 更新日期：2026-04-30（v2.0.0）

## 總覽

SyncTranslate 目前的主線架構：

- 音訊擷取與路由
- ASR v2（faster-whisper + silero-vad）
- 翻譯（LM Studio）
- TTS / passthrough
- UI / diagnostics / export

ASR 已完全收斂到 ASR v2。所有舊 pipeline 名稱在載入時會正規化至 v2 路徑。

主要程式入口：`main.py`

## 音訊資料流

```
AudioCapture（本地 / 遠端）
  └─> AudioRouter（通道分派）
        └─> ASRManagerV2（建立或重用 runtime）
              └─> EndpointingRuntime（VAD + soft/hard endpoint）
                    └─> SourceRuntimeV2（partial / final 決策）
                          └─> TranscriptPostProcessor（後處理）
                                └─> TranslationDispatcher（翻譯）
                                      └─> TTSDispatcher（TTS / passthrough）
                                            └─> UI（字幕顯示）
```

## 通道模型

專案固定使用兩條邏輯通道：`local` / `remote`

各通道有獨立的：ASR 語言、翻譯目標、TTS 聲音、queue、runtime stats

## ASR v2 架構

### 核心模組

| 模組 | 說明 |
|---|---|
| `app/infra/asr/manager_v2.py` | `ASRManagerV2`，建立 / 重用通道 runtime；含 `create_asr_manager()` factory |
| `app/infra/asr/profile_selection.py` | UI / config / runtime 共用的 ASR profile 入口 |
| `app/infra/asr/language_profiles.py` | 依語言調整 VAD / ASR 設定（品質調校層，不改變 backend）|
| `app/infra/asr/backend_resolution.py` | 語言 → backend 決策（`BackendResolution` dataclass）|
| `app/infra/asr/worker_v2.py` | `SourceRuntimeV2`，chunk queue / partial / final 主迴圈 |
| `app/infra/asr/backend_v2.py` | partial + final backend pair |
| `app/infra/asr/endpointing_v2.py` | `EndpointingRuntime`，VAD + soft/hard endpoint |
| `app/infra/asr/frontend_v2.py` | `AsrAudioFrontendV2`，音訊前處理（resampling / frontend chain）|
| `app/infra/asr/resampling.py` | 16k resampling，faster-whisper / silero-vad 共用 |
| `app/infra/asr/transcript_validator_v2.py` | final 事件驗證（hallucination / script mismatch 拒絕）|
| `app/infra/asr/faster_whisper_adapter.py` | process-level singleton，lazy init，CUDA fallback |
| `app/infra/asr/_hallucination_filter.py` | 幻覺過濾函數集中（silence hallucination / tail / loop 等）|
| `app/infra/asr/_adaptive_tuner.py` | `_AdaptiveTunerMixin`，adaptive length limit（SourceRuntimeV2 使用）|

### 模型自動路由

ASR 模型由系統自動路由，**不需手動選擇**：

| 通道語言 | ASR 模型 | Profile |
|---|---|---|
| `zh-TW` / `zh-CN` / `cmn` / `yue` | `runtimes/models/belle-zh-ct2` | `asr_profiles.chinese` |
| 其他語言 / `auto` | `large-v3-turbo` | `asr_profiles.non_chinese` |
| `none` | — | 停用 |

`language_profiles.py` 在 backend 建立前依語言調整 ASR 設定（屬品質調校層，不改變 backend 種類）：
- 中文：保守 VAD、`no_speech_threshold=0.25`、移除 previous-text conditioning
- 非中文 / auto：清掉中文 initial prompt，避免跨語言偏置

### Endpointing

`EndpointingRuntime` 輸出的重要訊號：
`speech_started` / `soft_endpoint` / `hard_endpoint` / `speech_probability` / `is_speech_frame`

worker_v2 用這些訊號累積 segment-local 統計，供 final validator 與幻覺過濾使用。

### Endpoint Profiles（`endpoint_profiles.py`）

透過 `runtime.asr_profile_local` / `asr_profile_remote` 選用：

| 名稱 | 適用場景 |
|---|---|
| `default` | 一般室內環境 |
| `meeting_room` | 遠端會議 / 廣播語音（會議模式預設）|
| `turn_taking` | 雙向短句對談（對話模式預設）|
| `headset` | 頭戴式麥克風近講 |
| `noisy_environment` | 嘈雜環境 |
| `max_accuracy` | 犧牲延遲換精度 |
| `low_latency` | 即時字幕嚴格延遲要求 |

### StreamingPolicy（`streaming_policy.py`）

三段 degradation 策略，根據 queue backlog 與 dropped_chunks 自動降級：
- `NORMAL` → `CONGESTED`（延長 partial interval 1.5×）→ `DEGRADED`（壓制 partial，3×）

## UI 層

### Local AI 頁（`app/ui/pages/local_ai_page.py`）

目前提供 2 種情境模式：
- **會議模式（穩定切段）**：`meeting_monitor`，VAD min_silence=600ms，soft_final=4000ms
- **對話模式（低延遲）**：`dialogue`，VAD min_silence=280ms，soft_final=2000ms

ASR 模型為唯讀 Label（固定自動路由，不可手動修改）。

### Live Caption 頁（`app/ui/pages/live_caption_page.py`）

config 靜態 helper 已移至 `app/ui/pages/_live_caption_config.py`。

### Diagnostics / Session Report

`logs/session_reports/*.json` 重要欄位：

- `resolved_backend`、`device_effective`、`model_init_mode`
- `speech_started_count`、`soft_endpoint_count`、`hard_endpoint_count`
- `partial_count`、`final_count`
- `postprocessor.final.rejected_count`、`postprocessor.final.last_rejection_reason`
- `dropped_chunks`
