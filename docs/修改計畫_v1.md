## 二、ASR 是否支援 48kHz 直送（免重採樣）

### 結論：**不可行，無需修改**

Faster-Whisper / OpenAI Whisper 的 encoder 在 C++ 層硬寫 `WHISPER_SAMPLE_RATE = 16000`。無論 Python 層傳入何種取樣率，`ctranslate2` 的 Whisper forward pass 假定輸入已是 16kHz log-mel。送入 48kHz 音訊不會報錯，但 mel-spectrogram 頻率軸會嚴重偏移，導致辨識率崩潰。

**現行流程已正確**：
```
AudioCapture (48kHz) → resample_audio(48k→16k) → faster_whisper_adapter → Whisper
```

`app/infra/asr/resampling.py` 中的整數倍降採樣路徑（`48000 / 16000 = 3`）使用 Hamming 低通濾波器，已是最佳化路徑。

### 建議動作

只需在文件中釐清此設計決策，避免未來誤改。可在 `docs/架構說明.md` 補充一節「ASR 取樣率說明」：
> 音訊擷取固定以 48kHz 進行以配合驅動格式。Whisper 引擎需要 16kHz 輸入，故有不可省略的重採樣步驟。此重採樣已在 `app/infra/asr/resampling.py` 以整數倍（3:1）Hamming 低通濾波器實現，延遲可忽略。

另需同步校正文案：`README.md` 目前將 bridge 描述成 `48kHz 48-bit ring buffer`，這是文件錯字，應改成 `48kHz PCM ring buffer` 或等價描述。

## 三、TTS 關閉時的直通音量太小與機械音問題

### 根本原因分析

先校正一點：`passthrough_gain` 目前已經接入 `TTSManager._apply_passthrough_gain()`，且有測試覆蓋增益與 clipping 行為，因此它不再是「設定存在但完全沒接線」的問題。

`app/infra/tts/playback_queue.py → TTSManager` 中「passthrough」模式（TTS voice = none 時的直通路徑）有以下問題：

**機械音原因（主要）**：
1. `_passthrough_gate_rms = 0.0055`：RMS 噪音閘門會在靜音片段強制靜音，導致語音啟動/結束處產生「咔嗒」聲。
2. `_passthrough_gate_until` / `_passthrough_warmup_until`：暖機與閘門超時會讓前 0.24s 靜音，產生開頭截斷。
3. `_passthrough_batch_wait_sec = 0.10` + `_passthrough_batch_min_chunks = 4`：批次等待機制造成 ~100ms 間歇性音頻停頓。
4. `_passthrough_backoff_until`：TTS 合成失敗後的退避靜音會影響直通路徑（兩者不該共用退避狀態）。

**音量太小原因**：
- `runtime.passthrough_gain` 已可正確影響送入 sink 的 passthrough 音訊；現況下較大的問題是 gate / warmup / batch / backoff 導致片段被延遲或略過，造成體感音量偏小、句首句尾被吃掉。
- 若 worker 存活但被 `_passthrough_gate_until`、`_passthrough_warmup_until` 或 `_passthrough_backoff_until` 抑制，音訊會被略過，這會比單純增益設定更直接影響聽感。
- TTS sink 的 `set_volume()` 和 passthrough gain 仍是獨立機制，用戶可能混淆；這點仍值得在 UI 或文件中釐清。

### 修改計畫

#### 3-A 新增「純直通模式」(true passthrough mode)

在 `TTSManager` 新增 output mode `"direct_passthrough"`，完全繞過所有閘門、暖機、批次邏輯：

```python
# playback_queue.py
VALID_OUTPUT_MODES = {"tts", "subtitle_only", "passthrough", "direct_passthrough"}

def submit_passthrough(self, channel: str, audio: np.ndarray, sample_rate: float) -> None:
    key = ...
    mode = self._output_mode.get(key)
    if mode == "direct_passthrough":
        # 完全不經過任何 gate/warmup/batch，直接送 sink
        gained = self._apply_passthrough_gain(audio)
        self._sinks[key].push_passthrough(audio=gained, sample_rate=float(sample_rate))
        return
    if mode != "passthrough":
        return
    # 原有帶 gate 的 passthrough 邏輯（保留給有 TTS 交互的場景）
    ...
```

#### 3-B 機械音 - 調整閘門預設值

```python
# playback_queue.py __init__ 預設值調整
self._passthrough_gate_rms = 0.0015       # 從 0.0055 降低，避免截斷低聲量語音
self._passthrough_warmup_sec = 0.08       # 從 0.24 縮短
self._passthrough_gate_timeout_sec = 0.20 # 從 0.60 縮短
self._passthrough_batch_wait_sec = 0.04   # 從 0.10 縮短
self._passthrough_batch_min_chunks = 2    # 從 4 降低
```

可改為從 `config.runtime` 讀取這些值，方便用戶調整：
```yaml
# config.yaml → runtime 段
passthrough_gate_rms: 0.0015
passthrough_warmup_sec: 0.08
```

#### 3-C TTS 退避狀態不應影響直通

```python
# _run_synth 失敗時：
self._passthrough_backoff_until[channel] = ...  # 只在 mode == "passthrough" 時設
# 新增檢查：
if self._output_mode.get(channel) in ("direct_passthrough",):
    return  # 不觸發退避
```

#### 3-D `config.yaml` 新增明確的直通增益設定

```yaml
runtime:
  passthrough_gain: 1.0           # 直通音量倍率（1.0 = 不增益）
  passthrough_mode: direct        # "direct" | "gated"（預設 direct 避免機械音）
```

#### 3-E `app/infra/config/schema.py → RuntimeConfig`

### 新語義定義

| 模式 | 啟用裝置 | 用途 | ASR | 翻譯 | TTS |

#### 4-B `app/application/audio_router.py → _desired_source_state()`

```python
def _desired_source_state(self) -> dict[str, dict[str, bool]]:
    runtime = self._effective_runtime
    session_mode = str(getattr(runtime, "session_mode", "meeting") or "meeting")
    
    remote_asr_lang = str(getattr(runtime, "remote_asr_language", "none") or "none")
    local_asr_lang = str(getattr(runtime, "local_asr_language", "none") or "none")
    
    if session_mode == "meeting":
        # 會議模式：只啟用 remote 捕捉（捕捉會議聲音）→ ASR → 翻譯 → 顯示字幕
        # 本地 mic 不啟動（不發言）
        return {
            "remote": {
                "capture": remote_asr_lang != "none",
                "asr": remote_asr_lang != "none",
            },
            "local": {
                "capture": False,
                "asr": False,
            },
        }
    else:  # dialogue
        # 對話模式：雙向，與現有邏輯相同
        return {
            "remote": {
                "capture": remote_asr_lang != "none" or self._tts_manager.is_passthrough_enabled("local"),
                "asr": remote_asr_lang != "none",
            },
            "local": {
                "capture": local_asr_lang != "none" or self._tts_manager.is_passthrough_enabled("remote"),
                "asr": local_asr_lang != "none",
            },
        }
```

## 五、LLM 翻譯跟隨 TTS 開關，TTS 關閉時改為真正直通

### 現況補充

目前實作中，是否啟用翻譯主要由 `translation_target != none` 與該 channel 的 ASR 是否啟用決定；`tts voice = none` 只會讓 channel 切到 `passthrough`，不會自動停用翻譯。因此本節描述的是「新的行為定義」，不是現況文件化。

### 邏輯定義

```
當 remote TTS 語音 = "none"（關閉）：
  → 遠端輸入的音訊不送 ASR，直接送本地輸出（真正直通）
  → 不需要 LLM 翻譯
  → remote original / remote translated 面板可隱藏或顯示「直通中」

當 local TTS 語音 = "none"（關閉）：
  → 本地輸入的音訊不送 ASR，直接送遠端輸出（真正直通）
  → 不需要 LLM 翻譯
  → local original / local translated 面板可隱藏或顯示「直通中」
```

### 修改點

#### 5-A `app/infra/config/schema.py → RuntimeConfig`

新增欄位：

#### 5-C `app/application/audio_router.py → _handle_source_audio_chunk()`

```python
def _handle_source_audio_chunk(self, *, source, passthrough_channel, chunk, sample_rate):
    self._state.tick()
    
    # 判斷此 source 是否為「真正直通」模式
    runtime = self._effective_runtime
    tts_voice_attr = f"{passthrough_channel}_tts_voice"
    tts_voice = str(getattr(runtime, tts_voice_attr, "none") or "none")
    true_passthrough = (tts_voice == "none")
    
    if true_passthrough:
        # 真正直通：跳過 ASR，直接送輸出
        # passthrough_channel = 要輸出到的 channel (local 或 remote)
        self._tts_manager.submit_direct_passthrough(passthrough_channel, chunk, sample_rate)
        return
    
    if self._state.can_accept_asr(source):
        self._asr_manager.submit(source, chunk, sample_rate)
    self._tts_manager.submit_passthrough(passthrough_channel, chunk, sample_rate)
```

> **注意**：真正直通時，`submit_direct_passthrough` 對應第三項的 `direct_passthrough` mode，完全繞過閘門。

#### 5-D `app/infra/tts/playback_queue.py`

新增 `submit_direct_passthrough()` 方法：
```python
def submit_direct_passthrough(self, channel: str, audio: np.ndarray, sample_rate: float) -> None:
    """真正直通：完全繞過 gate / warmup / batch，直接寫入 sink。"""
    key = channel if channel in ("local", "remote") else "local"
    if audio.size == 0:
        return
    try:
        gained = self._apply_passthrough_gain(audio)
        self._sinks[key].push_passthrough(audio=gained, sample_rate=float(sample_rate))
    except Exception as exc:
        if self._on_error:
            self._on_error(ErrorEvent(
                level="warning", module="tts_manager", source=key,
                code="direct_passthrough_failed",
                message="Direct passthrough failed", detail=str(exc),
            ))
```

#### 5-E `app/ui/pages/live_caption_page.py`

## 六、ASR 語言取消「自動」，改為五語言固定選單

### 現狀

`app/ui/pages/live_caption_page.py → _ASR_CHOICES`：
```python
_ASR_CHOICES: list[tuple[str, str]] = [
    ("none", "無"),
    ("auto", "自動"),   # ← 要移除
    ("zh-TW", "中文"),
    ("en", "英文"),
    ("ja", "日文"),
    ("ko", "韓文"),
    ("th", "泰文"),
]
```

### 移除「自動」的原因

- `auto` 在 `normalize_asr_language()` 直接回傳 `"auto"` 傳給 faster-whisper，由模型自行偵測語言，這會損耗 ~30ms 且容易在中英混雜時誤判。
- 五語言各有對應的 `LanguageAsrProfile`（zh/en/ja/ko/th），包含獨立的 VAD、endpoint、前置增強參數。
