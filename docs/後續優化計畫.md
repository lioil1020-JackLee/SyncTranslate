### 2-C：`AudioRouter` 實際使用 `ASREventProcessor`（消除重疊）
**風險：中 ｜ 預估時間：2-3 小時**

#### 現況（真正的架構問題）
`app/application/asr_event_processor.py` 的文件說明：
> "AudioRouter delegates to this class after receiving a raw ASREventWithSource"

但 `AudioRouter._handle_asr_event_payload` **並沒有委派給它**，而是自己重新實作了相同的邏輯：

| 步驟 | `ASREventProcessor.handle()` | `AudioRouter._handle_asr_event_payload()` |
|------|-------------------------------|------------------------------------------|
| 延遲閘（drop if too old） | ✅ `_should_drop()` | ✅ `_should_drop_over_latency()` |
| 文字後處理 | ✅ `postprocessor.process_final/partial` | ✅ 相同邏輯 |
| 儲存原始 transcript | ✅ `_store()` | ✅ `_maybe_store_transcript()` |
| 通知 callback | ✅ `_on_store()` | ❌ 沒有（emit 是分開的） |
| 翻譯/TTS 決策 | ❌（不處理） | ✅ 由 `AudioRouter` 繼續處理 |

**兩套邏輯並存，`ASREventProcessor` 未被使用，屬於死代碼。**

#### 修正選項

**選項 A（推薦）：讓 AudioRouter 使用 ASREventProcessor**
```python
# audio_router.py __init__ 新增
self._asr_event_processor = ASREventProcessor(
    transcript_buffer=transcript_buffer,
    postprocessor=self._postprocessor,
    translator_manager=translator_manager,
)

# _handle_asr_event_payload 精簡為：
def _handle_asr_event_payload(self, event):
    self._emit_asr_event(event)
    self._latency_tracker.record_asr(event)
    stored = self._asr_event_processor.handle(event)
    if not stored:
        return
    # 繼續翻譯/TTS 決策...
```

**選項 B：刪除 ASREventProcessor（死代碼清理）**
若 `ASREventProcessor` 沒有被任何地方使用，直接刪除，保持代碼乾淨。

#### 需先確認
執行 `grep -r "ASREventProcessor" tests/` 確認是否有測試直接使用它。
若有測試 → 選項 A（整合），若無測試且無外部使用 → 選項 B（刪除）。

### 2-E：`audio_router.py` 的 `getattr` 防禦性存取改善
**風險：低 ｜ 預估時間：1 小時**

#### 現況
`audio_router.py` 中多處出現雙層 `getattr`：
```python
runtime = getattr(self._runtime_config, "runtime", None)
value = getattr(runtime, "some_field", default) if runtime is not None else default
```

根本原因：`self._runtime_config` 在 `__init__` 時為 `None`，在 `start()` 前不保證被設定。

#### 改善方案
新增一個 typed property：
```python
@property
def _effective_runtime(self) -> RuntimeConfig | None:
    cfg = self._runtime_config
    return cfg.runtime if cfg is not None else None
```

各處改為：
```python
rt = self._effective_runtime
value = rt.some_field if rt is not None else default
```

這樣更清晰，也讓 type checker 有更好的推斷。
