# 音訊裝置建議配置

這份文件提供一套在 Windows + Voicemeeter + 會議軟體環境下較穩定的 SyncTranslate 路由方式。

## 建議設備對應

### 會議軟體

- 喇叭：`Voicemeeter AUX Input`
- 麥克風：`Voicemeeter Out B1`

### SyncTranslate

- 遠端輸入：`Voicemeeter Out B2`
- 遠端輸出：`Voicemeeter Input`
- 本地輸入：你的實體麥克風
- 本地輸出：你的耳機或喇叭

## Voicemeeter 建議

### `Voicemeeter AUX Input`

這一路通常接會議軟體的聲音。

建議：

- 開 `B2`
- 不開 `B1`

用途是把會議端聲音送進 SyncTranslate 的遠端輸入。

### `Voicemeeter Input`

這一路通常接 SyncTranslate 的遠端輸出。

建議：

- 開 `B1`
- 不開 `B2`

用途是把 SyncTranslate 要送回會議端的聲音交給會議軟體麥克風。

### 實體麥克風輸入

建議直接進 SyncTranslate，不要再繞一層 Voicemeeter，除非你真的需要：

- 額外的 EQ / gate / comp
- 多路混音
- 特殊監聽配置

## 推薦資料流

### 遠端來音

`會議軟體 -> Voicemeeter AUX Input -> B2 -> SyncTranslate 遠端輸入`

### 本地送出

`實體麥克風 -> SyncTranslate 本地輸入`

### 送回會議

`SyncTranslate 遠端輸出 -> Voicemeeter Input -> B1 -> 會議軟體麥克風`

## 為什麼不要把本地麥克風先送進 Voicemeeter

如果只是做即時翻譯，直接把實體麥克風交給 SyncTranslate 通常比較穩，原因是：

- 少一層額外 gain
- 少一層 bus 路由
- 比較不容易產生回授或重複送音

## 常見問題

### 對方聽不到我

先檢查：

1. `本地輸入` 是否選到真正的麥克風
2. `遠端輸出` 是否選到 `Voicemeeter Input`
3. `Voicemeeter Input` strip 是否有開 `B1`
4. 會議軟體麥克風是否真的選 `Voicemeeter Out B1`

### 對方聽到的是翻譯，不是原聲

這通常不是路由錯，而是即時字幕頁該方向的 `模式` 正在使用 `翻譯`。

- 要送原聲：切 `直通`
- 要送翻譯語音：切 `翻譯`

### 聲音太小或失真

先把 SyncTranslate 的 `輸出音量` 保守設在 `1.2x ~ 1.6x`。

若拉太高，虛擬音源與會議軟體常會因失真或降噪而把聲音吃掉。
