# 音訊裝置建議配置

SyncTranslate 同時處理本機與遠端兩個方向的音訊。正確的裝置配置會直接影響 ASR 是否乾淨、TTS 是否回到正確的人、以及是否產生回音。

## 通道概念

- `local input`：你的麥克風。
- `remote input`：會議軟體或系統播放聲音。
- `local output`：你要聽到遠端翻譯的播放裝置。
- `remote output`：要送回會議軟體的播放裝置，通常是虛擬輸入裝置。

TTS 輸出會交叉：

- 遠端說話的翻譯播放到 `local output`。
- 本機說話的翻譯播放到 `remote output`。

## 推薦配置

### 使用 Voicemeeter / VB-CABLE

這是最穩定的會議雙向配置。

| SyncTranslate 欄位 | 建議裝置 |
| --- | --- |
| local input | 實體麥克風 |
| remote input | 會議軟體輸出或 Voicemeeter bus |
| local output | 耳機或喇叭 |
| remote output | VB-CABLE / Voicemeeter virtual input |

會議軟體中：

- 麥克風選擇 SyncTranslate 的 `remote output` 對應虛擬裝置。
- 喇叭輸出到 SyncTranslate 的 `remote input` 來源。

### 只看字幕

若只需要字幕與翻譯，不需要把 TTS 送回會議：

- local input 設成麥克風。
- remote input 設成會議音訊。
- local output 可設耳機。
- remote output 可留空或設成不使用。
- TTS voice 選 `無` 或輸出模式使用 `subtitle_only`。

### 單向翻譯

若只翻譯遠端語音給自己聽：

- 啟用 remote ASR。
- local ASR 可設為 none。
- remote target 設為你的目標語言。
- local output 設為耳機。
- remote output 可不用。

## 避免回音

- 優先用耳機，不要讓喇叭聲音被麥克風重新收進去。
- remote output 不要選到你自己的喇叭。
- 會議軟體的麥克風不要同時收實體麥克風與 SyncTranslate 輸出。
- 若使用 passthrough，確認只有一條音訊路徑送回會議。

## 裝置名稱選擇

UI 會列出 host API 與裝置名稱。Windows 上常見 host API 包含：

- WASAPI
- MME
- DirectSound

通常優先使用 WASAPI。若虛擬裝置在 WASAPI 下不穩，再改試 MME 或 DirectSound。

## 音量與增益

- input gain 只補足太小的輸入，不建議拉太高。
- output volume 控制播放端音量。
- Windows endpoint volume 會影響實際裝置音量。
- ASR 若常出現亂字，先降低遠端輸入增益或關閉會議背景音樂。

## 檢查順序

1. 在設定頁確認四個裝置都有選對。
2. 先開只顯示字幕，確認 ASR 文字穩定。
3. 再開翻譯，確認 target language 正確。
4. 最後開 TTS，確認聲音進到正確輸出。
5. 若對方聽到回音，優先檢查 remote output 與會議軟體麥克風設定。
