# 音訊裝置建議配置

本文對齊目前 `AudioRouteConfig` 與 UI 命名。SyncTranslate 有兩個輸入與兩個輸出：

| 設定 key | 用途 |
| --- | --- |
| `audio.meeting_in` | 遠端/會議音訊輸入 |
| `audio.microphone_in` | 本地音訊輸入 |
| `audio.speaker_out` | 本地喇叭輸出 |
| `audio.meeting_out` | 送回會議端的輸出 |

音量 key：

| 設定 key | 用途 |
| --- | --- |
| `audio.meeting_in_gain` | 遠端輸入 gain |
| `audio.microphone_in_gain` | 本地輸入 gain |
| `audio.speaker_out_volume` | 本地輸出 volume |
| `audio.meeting_out_volume` | 會議輸出 volume |

## 建議裝置

Windows 建議使用 WASAPI。若使用 Voicemeeter 或 VB-CABLE，請固定路由，避免 local/remote 同時收到同一個混音 bus。

常見配置：

| SyncTranslate | 建議裝置 |
| --- | --- |
| `meeting_in` | 會議軟體或系統播放輸出的 virtual output |
| `microphone_in` | 麥克風或本地要辨識的 virtual output |
| `speaker_out` | 你自己要聽到的喇叭/耳機 |
| `meeting_out` | 送回會議軟體的 virtual input |

## Voicemeeter 範例

一種常見配置：

```yaml
audio:
  meeting_in: Windows WASAPI::Voicemeeter Out B1 (VB-Audio Voicemeeter VAIO)
  microphone_in: Windows WASAPI::Voicemeeter Out B2 (VB-Audio Voicemeeter VAIO)
  speaker_out: Windows WASAPI::Voicemeeter In 1 (VB-Audio Voicemeeter VAIO)
  meeting_out: Windows WASAPI::Voicemeeter AUX Input (VB-Audio Voicemeeter VAIO)
```

請確認：

- 會議軟體輸出不要同時混進 `microphone_in`。
- 本地麥克風不要同時混進 `meeting_in`。
- 若啟用 passthrough，確認不會形成回授。

## 模式建議

### 只看字幕

- TTS voice 設為 `none`。
- `runtime.tts_output_mode` 可用 `subtitle_only` 或 `passthrough`。
- 只需要 remote 字幕時，可將 `runtime.local_asr_language` 設為 `none`。
- 只需要 local 字幕時，可將 `runtime.remote_asr_language` 設為 `none`。

### 雙向字幕

- `runtime.local_asr_language` 設為本地語言，例如 `zh-TW`。
- `runtime.remote_asr_language` 可設 `auto` 或明確語言。
- local/remote 都建議使用 `meeting_room` profile。
- `runtime.asr_queue_maxsize_local` / `remote` 建議至少 `256`。

### 低延遲對話

`turn_taking` 會降低 final 延遲，但會增加 ASR decode 頻率。設定 UI 會把 ASR queue 保持在至少 `256`，worker 也會在高壓時暫停 partial；若同時跑雙 ASR 與本地 LLM，仍可能更容易 queue overflow。

建議：

- 先使用 `meeting_room`。
- 確認沒有 `ASR queue overflow` 後，再測 `turn_taking`。
- 若 queue overflow 持續增加，改回 `meeting_room`。中文 ASR 目前預設 `large-v3-turbo`；若特定素材 belle-zh-ct2 較準，可在設定 UI 針對該場景切換。

## 音量與 ASR 品質

ASR frontend 會做基本 AGC，但輸入品質仍很重要。

建議：

- 輸入音量不要太小，否則 AGC 會放大底噪。
- 避免 clipping。
- 避免背景音樂與人聲混在同一路。
- local/remote capture level 若長時間幾乎相同，要檢查是否兩路接到同一個混音來源。

session report 可檢查：

```text
stats.capture.local.level
stats.capture.remote.level
stats.asr.local.frontend.input_rms
stats.asr.local.frontend.output_rms
stats.asr.local.frontend.speech_ratio
```

## queue overflow 排查

看到：

```text
ASR queue overflow (local): dropped 11 chunks total
```

代表 local ASR 已丟音訊。排查順序：

1. 確認 `runtime.asr_queue_maxsize_local` 是 `256`。
2. 確認 `runtime.asr_profile_local` 是 `meeting_room`。
3. 若只需要本地中文，暫時將 `runtime.remote_asr_language` 設為 `none`。
4. 降低 TTS 或翻譯負載。
5. 檢查 GPU 是否同時被其他程式占用。
6. 匯出 session report，查看 `stats.asr.local.dropped_chunks` 是否持續增加。
7. 查看 UI 診斷列：`q=current/max`、`drop`、`deg`、`model`、`profile`、`enhancement` 會直接反映目前 ASR 負載與實際模型。診斷列也會顯示以下額外狀態：
   - `fp=on`：final priority 模式已啟用，代表 queue 壓力或 final 延遲觸發了 partial 壓制，系統正在優先保護 final 輸出。這不是錯誤，但若持續顯示，代表整體負載偏高。
   - `lang=auto→zh-TW`：auto 語言切換已生效，effective profile 已切到中文。若顯示 `(pending)` 代表切換排程中、runtime 尚未重建完成。
8. 若診斷列顯示 `capture=possible-same-source`，代表 local/remote 可能收到同一路音訊，需要重新檢查 Voicemeeter/VB-CABLE 路由。

偶發 `dropped 1` 可以觀察；持續跳 `11、21、31...` 就需要降低負載或調整配置。

## 避免回音

若啟用 TTS 或 passthrough：

- 不要讓 `speaker_out` 再被 `microphone_in` 收進來。
- 不要讓 `meeting_out` 再被 `meeting_in` 收進來。
- 使用耳機比喇叭穩定。
- 必要時啟用 echo guard 設定。

相關 runtime key：

- `local_echo_guard_enabled`
- `local_echo_guard_resume_delay_ms`
- `remote_echo_guard_resume_delay_ms`
