# SyncTranslate 首次啟動引導

## 前置條件

- Windows 10/11（64-bit）
- Administrator 權限（用於驅動安裝）
- 至少 8GB RAM（推薦 16GB 以上以運行 AI 模型）
- 可選：NVIDIA/AMD 顯卡（用於 ASR/翻譯加速）

## 步驟 1：驅動安裝（首次只需一次）

### 1.1 啟用 Windows Test Mode

test-signed driver 需要 Test Mode。以 **Administrator** 身份開啟 PowerShell，執行：

```powershell
Set-Location E:\py\SyncTranslate
.\drivers\synctranslate_virtual_audio\scripts\enable_test_mode.ps1
```

腳本會提示重新開機。**系統會自動重新開機一次**。

重新開機後，檢查 Secure Boot：
- 若遇到「值受安全開機原則保護」錯誤，執行診斷：
  ```powershell
  .\drivers\synctranslate_virtual_audio\scripts\diagnose_test_signing.ps1
  ```
- 若 Secure Boot 仍啟用，進 BIOS 關閉它（若有 BitLocker 先暫停）

### 1.2 安裝測試憑證

驅動使用測試簽章，需要信任測試憑證。以 **Administrator** 身份執行：

```powershell
$certPath = "artifacts\driver\synctranslate_virtual_audio\SyncTranslateVirtualAudioTest.cer"
Import-Certificate -FilePath $certPath -CertStoreLocation Cert:\LocalMachine\Root
Import-Certificate -FilePath $certPath -CertStoreLocation Cert:\LocalMachine\TrustedPublisher
```

### 1.3 安裝 MSI

以 **Administrator** 身份執行：

```powershell
msiexec /i "artifacts/driver/synctranslate_virtual_audio/SyncTranslateVirtualAudioDriver.msi"
```

或直接雙擊 MSI 檔案（Windows 會自動提升權限）。

MSI 安裝完成後會**自動建立 SyncTranslate 虛擬音訊裝置**（無需手動操作）。

### 1.4 驗證驅動安裝

重新開機後，檢查設備是否可見：

1. 開啟「聲音設定」（右下角音量圖示 → 聲音設定）
2. 向下滾動至「進階」→「所有音訊設備」
3. 確認能看到：
   - `SyncTranslate Virtual Speaker`
   - `SyncTranslate Virtual Microphone`

若看不到，檢查「裝置管理員」→「聲音、視訊和遊戲控制器」確認驅動狀態。

## 步驟 2：首次運行 App

### 2.1 準備環境（開發版）

若從原始碼開發：

```powershell
cd E:\py\SyncTranslate
uv sync
uv run python main.py --check
```

`--check` 會驗證配置和 runtime 是否就緒。

### 2.2 啟動 App

```powershell
uv run python main.py
```

或直接：

```powershell
python main.py
```

**首次啟動時**，App 會自動：
1. 偵測虛擬音訊設備
2. 啟動 bridge process（`sync_audio_bridge.exe`）
3. 初始化 ASR 和翻譯模型（首次下載約 2-3GB，需等待）

UI 會顯示：
- 虛擬設備摘要
- 橋接連接狀態
- ASR/翻譯模型狀態

### 2.3 基本配置

在 UI 設定分頁中設定：

| 欄位 | 選項 | 說明 |
|-----|------|------|
| **遠端語言** | `auto` / `en` / `zh-TW` / ... | 會議/遠端音訊語言 |
| **本地語言** | `zh-TW` / `en` / ... | 麥克風輸入語言 |
| **翻譯方向** | `EN → ZH-TW` / `ZH-TW → EN` | 雙向翻譯 |
| **ASR 準確度** | `low_latency` / `balanced` / `high_accuracy` | 速度 vs. 準確度 |

## 步驟 3：配置通訊軟體

### 3.1 Zoom

1. 開啟 Zoom 設定 → **音訊** 
2. **麥克風**：選擇 `SyncTranslate Virtual Microphone`
3. **喇叭**：選擇 `SyncTranslate Virtual Speaker`，若 Windows 顯示為 `喇叭 (SyncTranslate Virtual Audio Device)` 則選該端點
4. 點擊「測試麥克風」確認聲音輸入

### 3.2 Teams

1. 開啟 Teams 設定 → **裝置** → **音訊設備**
2. **麥克風**：選擇 `SyncTranslate Virtual Microphone`
3. **喇叭**：選擇 `SyncTranslate Virtual Speaker`，若 Windows 顯示為 `喇叭 (SyncTranslate Virtual Audio Device)` 則選該端點
4. 點擊「進行測試通話」

### 3.3 Google Meet

1. 進入 Meet 通話 → 左下角「設定」⚙️
2. **麥克風**：`SyncTranslate Virtual Microphone`
3. **喇叭**：`SyncTranslate Virtual Speaker`，或 `喇叭 (SyncTranslate Virtual Audio Device)`

### 3.4 LINE、WhatsApp、Discord

類似步驟。在各應用的設定 → 音訊/麥克風/喇叭中選擇 SyncTranslate 虛擬設備。

## 步驟 4：測試

### 4.1 快速測試（~2 分鐘）

```powershell
.\Tools\matrix\phase5_phase6_quick_gate.py --soak-preset smoke
```

這會執行：
- 軟體矩陣測試：所有 6 個通訊軟體
- 穩定性測試：120 秒 soak

若全部通過（`overall_ok=true`），系統就緒。

### 4.2 完整測試（30 分鐘）

```powershell
.\tools\matrix\runtime_soak_2hr.py --preset medium
```

### 4.3 診斷信息

若有問題，查看 UI 診斷面板，記住以下關鍵指標：

- `router_running`：應為 `true`
- `bridge.connected`：應為 `true`
- `remote_input_running`：通話時應為 `true`
- `virtual_mic_dropped_frames`：應保持穩定（不應快速增長）
- `sink_backpressure_flushes`：控制機制動作的指標（正常）

## 故障排除

### 虛擬設備不可見

1. 檢查驅動是否成功安裝：在「裝置管理員」搜尋 SyncTranslate
2. 若驅動存在但設備不可見，重新開機系統
3. 若驅動本身不存在，重新執行 MSI 安裝程序

### App 啟動緩慢

首次啟動時會下載 AI 模型（~2-3GB），請等待。之後啟動會快速許多。

可在 `runtimes/models/` 目錄查看下載進度。

### 沒有聲音或翻譯

1. 確認通訊軟體已將麥克風/喇叭設為 SyncTranslate 虛擬設備
2. 檢查 UI 診斷面板，確認 `bridge.connected=true` 且 `remote_input_running=true`
3. 檢查設定中 `call_translation` 的各項播報開關

### 系統卡頓

- 檢查 CPU/GPU 使用率（ASR 模型很吃資源）
- 嘗試改用更快的 ASR 模型（如 `turbo`）
- 禁用不必要的前端處理（ASR enhancement）

### 需要更多幫助

請收集以下檔案提交問題報告：
- `logs/session_reports/` 下最新的 JSON 報告
- 完整的錯誤訊息截圖
- 系統資訊（Windows 版本、GPU 等）
