# 代碼修改：主包完整外部化（可用版）

## 1. 目標

本次改造目標不是縮減功能，而是把 AI 重依賴從主 GUI 包完整外部化，形成「可執行、可維護、可發佈」的正式架構：

- 主程式（SyncTranslate.exe）專注 UI / 音訊路由 / 協調
- 中文 ASR（FunASR）改為外部 runtime
- 非中文 ASR（faster-whisper）改為外部 runtime
- 共用 CUDA / Torch / ModelScope 依賴改為 shared runtime
- 打包時輸出成 runtimes 目錄，不再塞入主包核心路徑
- GitHub Release 允許超大產物分片上傳

---

## 2. 完成定義（Definition of Done）

### 2.1 執行面

- 程式啟動時會自動掛載外部 runtime site-packages
- 若缺少 FunASR 或 faster-whisper，錯誤訊息會明確指向外部 runtime 位置
- 主程式可在 frozen 模式與開發模式都正確解析 external runtime 路徑

### 2.2 打包面

- PyInstaller onedir 輸出包含 `runtimes/` 結構
- AI 重依賴（torch / torchaudio / funasr / faster-whisper / modelscope / onnxruntime / ctranslate2）不再進主包分析依賴
- release 資產若超過 2GB，自動分片，避免 gh-release 驗證失敗

### 2.3 維運面

- 提供可重建外部 runtime 的腳本
- 文件明確描述目錄結構、安裝、驗證、故障排除

## 4. 本次已修改的核心程式碼

### 4.1 啟動期掛載外部 runtime

- 新增：`app/bootstrap/external_runtime.py`
- 說明：
  - 啟動時掃描 `runtimes/shared`、`runtimes/funasr`、`runtimes/faster_whisper`
  - 自動加入 `site-packages` 到 `sys.path`
  - Windows 自動加入 DLL 搜尋目錄（torch/lib、onnxruntime/capi、nvidia/*/bin）
  - 若存在 `models` 目錄，自動設定 `MODELSCOPE_CACHE` / `HF_HOME`

### 4.2 主入口改為先掛載 runtime 再啟動 App

- 修改：`main.py`
- 說明：
  - 在 import `app_factory` 前呼叫 `configure_external_ai_runtime()`
  - 確保所有延遲載入的 ASR 模組可直接解析外部依賴

### 4.3 PyInstaller 規則改為外部化輸出

- 修改：`SyncTranslate-onedir.spec`
- 說明：
  - 新增 `_collect_external_runtime_packages()`
  - 將重依賴改輸出到 `runtimes/*/site-packages`
  - Analysis `excludes` 排除 AI 重依賴，避免塞回主包
  - 將 `tools/runtime_setup/*.ps1` 一併納入發行內容

### 4.4 錯誤訊息改成外部化導向

- 修改：`app/infra/asr/faster_whisper_adapter.py`
- 說明：缺依賴時提示「可用 uv 安裝或提供 runtimes/faster_whisper + runtimes/shared」

- 修改：`app/infra/asr/funasr_registry.py`
- 說明：缺依賴時提示「可用 uv 安裝或提供 runtimes/funasr + runtimes/shared」

### 4.5 外部 runtime 建置腳本

- 新增：`tools/runtime_setup/prepare_external_runtimes.ps1`
- 說明：
  - 一鍵建立 `shared` / `faster_whisper` / `funasr` 三套 runtime venv
  - 支援 `-CpuOnly` 模式
  - 可作為 CI 或本機釋出前步驟

### 4.6 Release 超大資產分片

- 修改：`.github/workflows/build-release.yml`
- 說明：
  - 打包流程現已：安裝依賴 → 準備外部 runtime → PyInstaller 構建 → 複製 runtimes → 壓縮
  - 若 zip 超過 2GB，自動切分成 `.part001/.part002...` 各 1900MB
  - 自動移除超大原始 zip，避免 gh-release asset size 驗證失敗
  - 追加 `rebuild-zip-from-parts.ps1` 供使用者重組

### 4.8 pyproject.toml 輕量化

- 修改：`pyproject.toml`
- 說明：
  - 移除 `[project.optional-dependencies] local = [...]`
  - 移除 `[tool.uv.sources]` 中的 pytorch-cu128 索引定義
  - 主套件依賴現已完全輕量（不含 torch / funasr / faster-whisper）
  - 用戶需透過 `prepare_external_runtimes.ps1` 管理 AI 運行時

### 4.9 uv.lock 重新生成

- 修改：`uv.lock`
- 說明：
  - 移除所有 AI 相關套件（torch / funasr / faster-whisper / ctranslate2 / tiktoken 等）
  - 保留主應用依賴（PySide6 / soundcard / pytest / pyinstaller）
  - 構建時只需 `uv sync --locked` + `prepare_external_runtimes.ps1`

## 5. 建置與打包流程（正式版）

### 5.1 本機準備

```powershell
# 主程式依賴（輕量級，不含 AI 套件）
uv sync --locked

# 準備外部 AI 運行時
powershell -ExecutionPolicy Bypass -File .\tools\runtime_setup\prepare_external_runtimes.ps1
```

### 5.2 建立外部 runtime（已在 5.1 執行）

CPU-only 版本（無 CUDA）：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\runtime_setup\prepare_external_runtimes.ps1 -CpuOnly
```

### 5.3 安裝 Build 工具

```powershell
uv sync --locked --group build
```

### 5.4 打包 onedir

```powershell
uv run pyinstaller .\SyncTranslate-onedir.spec --noconfirm --clean
```

### 5.5 複製外部 runtime 到 dist

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\runtime_setup\relocate_ai_runtime_artifacts.ps1
```

### 5.6 打包後自檢

```powershell
# 檢查外部 runtime 目錄是否存在
Get-ChildItem .\dist\SyncTranslate-onedir\runtimes -Recurse -Depth 2

# 啟動基本健康檢查
.\dist\SyncTranslate-onedir\SyncTranslate.exe --check
```

## 6. 風險與對策

### 6.1 風險：外部 runtime 缺套件

- 症狀：啟動後進入 ASR 時報 import error
- 對策：重跑 `prepare_external_runtimes.ps1` 或補齊對應 runtime 套件

### 6.2 風險：CUDA DLL 未解析

- 症狀：設定 cuda 但 runtime fallback 到 cpu
- 對策：
  - 確認 `runtimes/shared/site-packages/torch/lib` 與 `nvidia/*/bin` 存在
  - 確認 `torch.cuda.is_available()`

### 6.3 風險：release 資產過大

- 對策：workflow 已改為自動分片；使用者可用附帶腳本重組
