# 直通模式排錯清單

本清單用來排查以下情境：

- SyncTranslate 已啟動
- `遠端輸出 翻譯目標 = 無`
- `本地輸出 翻譯目標 = 無`
- `TTS語音 = 無`
- 預期應進入雙向 `passthrough`
- 但雙方仍聽不到對方

## 1. 先確認是不是啟動後的設定

必須先看到：

- 右上角按鈕變成 `停止`
- 左下角狀態列出現 `Session started: bidirectional`
- 四個面板右上的狀態點為綠色 `已啟動`

若還沒啟動，`passthrough` 不會工作。

## 2. 先確認 SyncTranslate 四格裝置

建議值：

- `遠端輸入` = `Voicemeeter Out B2`
- `遠端輸出` = `Voicemeeter Input`
- `本地輸入` = 實體麥克風
- `本地輸出` = 你的耳機 / 喇叭

常見錯誤：

- `本地輸入` 選成耳機或喇叭
- `遠端輸出` 選成 `Voicemeeter Out B1/B2`
- `本地輸出` 選成 Voicemeeter bus，而不是實際播放裝置

## 3. 先確認 UI 是否真的進入 passthrough

在目前程式裡：

- 翻譯目標 = `無`
- 對應 TTS 語音 = `無`

該方向 output mode 會被推導成 `passthrough`。

也就是：

- `遠端輸出 翻譯目標 = 無` -> `remote -> local` 直通
- `本地輸出 翻譯目標 = 無` -> `local -> remote` 直通

## 4. Voicemeeter 應該亮的按鈕

### `Voicemeeter AUX Input`

這是 LINE 喇叭進來的那條。

應該：

- 開 `B2`
- 不開 `B1`

### `Voicemeeter Input`

這是 SyncTranslate `遠端輸出` 送進來的那條。

應該：

- 開 `B1`
- 不開 `B2`

### `Stereo Input 1~3`

若 SyncTranslate 已直接抓實體麥克風，建議：

- 不要再把同一支實體麥克風掛到這些 strip

## 5. LINE 應該怎麼設

- 喇叭：`Voicemeeter AUX Input`
- 麥克風：`Voicemeeter Out B1`

## 6. 四步驟逐段驗證

### A. 驗證 LINE -> Voicemeeter

請對方說話，觀察：

- `Voicemeeter AUX Input` strip 是否有電平

若沒有：

- LINE 喇叭裝置選錯
- 或 LINE 根本沒輸出到 Voicemeeter

### B. 驗證 Voicemeeter -> SyncTranslate 遠端輸入

請對方說話，觀察：

- `B2` master 是否有電平
- SyncTranslate 左上 `遠端原文` 是否開始出現內容

若 `AUX Input` 有電平但 `B2` 沒有：

- `AUX Input` strip 沒開 `B2`

若 `B2` 有電平但 `遠端原文` 沒內容：

- `遠端輸入` 不是 `Voicemeeter Out B2`
- ASR 語言或裝置選錯

### C. 驗證 SyncTranslate 本地輸入

你自己說話，觀察：

- SyncTranslate 左下 `本地原文` 是否開始出現內容

若沒有：

- `本地輸入` 選錯
- 最常見是誤選成耳機 / 喇叭，而不是實體麥克風

### D. 驗證 SyncTranslate -> Voicemeeter -> LINE

你自己說話，觀察：

- `Voicemeeter Input` strip 是否有電平
- `B1` master 是否有電平
- LINE 麥克風電平是否有反應

若 `Voicemeeter Input` 沒電平：

- `遠端輸出` 選錯
- 或 SyncTranslate 根本沒有收到你的本地輸入

若 `Voicemeeter Input` 有電平但 `B1` 沒電平：

- `Voicemeeter Input` strip 沒開 `B1`

若 `B1` 有電平但 LINE 麥克風沒反應：

- LINE 麥克風不是 `Voicemeeter Out B1`

## 7. 目前程式中的方向定義

目前程式邏輯已驗證：

- `local` source 直通會送到 `remote` output channel
- `remote` source 直通會送到 `local` output channel

也就是：

- 你的麥克風會直通到 `遠端輸出`
- 對方會議聲會直通到 `本地輸出`

## 9. 目前最可疑的設定點

若你看到：

- 已啟動
- health check 正常
- 但完全沒聲音

最常見不是程式沒切成 passthrough，而是以下其中一項：

- `本地輸入` 選成耳機
- `AUX Input` 沒開 `B2`
- `Voicemeeter Input` 沒開 `B1`
- LINE 喇叭 / 麥克風裝置選錯
- `本地輸出` 不是你實際在聽的那個耳機 / 喇叭
