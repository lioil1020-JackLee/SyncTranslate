# 直通模式排錯清單

適用情境：

- 你已經把某個方向切成 `直通`
- 字幕可能正常，但對方或你自己仍然聽不到原聲

## 先確認 UI 狀態

1. 進入 `即時字幕`
2. 找到對應方向的翻譯區
3. 確認 `模式` 是 `直通`
4. 確認 `輸出音量` 不要太高，建議先用 `1.4x`

補充：

- 現在 `翻譯目標` 不再負責切模式
- 不需要把 `翻譯目標` 或 `TTS語音` 改成 `無` 來直通

## 再確認音訊路由

### 遠端聲音要播到本地

檢查：

- `遠端輸入` 是否選到會議聲音來源
- `本地輸出` 是否選到耳機 / 喇叭

### 本地聲音要送到遠端

檢查：

- `本地輸入` 是否選到真正的麥克風
- `遠端輸出` 是否選到會議虛擬麥克風

## Voicemeeter 使用者額外檢查

### `Voicemeeter AUX Input`

如果這一路承接會議軟體喇叭，請確認：

- 有開 `B2`
- 沒誤開其他不需要的 bus

### `Voicemeeter Input`

如果這一路承接 SyncTranslate 輸出，請確認：

- 有開 `B1`
- 沒把 `B2` 也一起打開造成回送

## 判斷是路由問題還是模式問題

### 情況 A：字幕有動，但完全沒聲音

優先懷疑：

- 輸出裝置選錯
- Voicemeeter bus 沒開
- 會議軟體沒有選對虛擬裝置

### 情況 B：聽到翻譯，不是原聲

優先懷疑：

- 該方向模式其實還在 `翻譯`

### 情況 C：偶爾有聲、偶爾沒聲

優先懷疑：

- 輸出音量過高造成失真
- 會議軟體降噪或自動音量抑制
- 虛擬音源鏈過長

## 建議檢查順序

1. 模式是否為 `直通`
2. `輸出音量` 先回到 `1.4x`
3. 輸入與輸出裝置是否正確
4. 若有 Voicemeeter，檢查 `B1 / B2`
5. 會議軟體的喇叭與麥克風是否選對
6. 重新開始一次 session

## 執行中熱切換補充

- 按下 `開始` 之後，`直通 / 翻譯`、翻譯目標、TTS 聲線、ASR 語言都可以直接熱切換。
- 切換後通常會從下一段或下一批音訊開始生效；已經在播放中的那一小段音訊不會被強制重算。
