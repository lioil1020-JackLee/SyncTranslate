# 安裝說明

本文件以 Windows 為主，從空白環境開始，帶你完成：

1. 安裝 LM Studio
2. 下載並載入可用的 LLM
3. 安裝 SyncTranslate
4. 準備音訊裝置
5. 完成第一次啟動

## 先決條件

- 作業系統：Windows 10 / 11 x64
- CPU：支援 AVX2
- RAM：建議至少 16 GB
- GPU：建議至少 4 GB 獨立 VRAM
- 網路：首次下載 LM Studio、LLM 與 Whisper 權重時需要

## 第 1 步：安裝 LM Studio

1. 到官方下載頁下載 Windows 版 LM Studio。
2. 安裝完成後啟動 LM Studio。

官方參考：

- https://lmstudio.ai/download
- https://lmstudio.ai/docs/app/system-requirements

## 第 2 步：在 LM Studio 下載模型

SyncTranslate 目前最建議先裝一個通用、穩定、可多語翻譯的模型。

### 建議模型

如果你的電腦屬於一般主流等級，建議優先選：

- `Qwen/Qwen3-4B-GGUF`
- 量化檔建議：`Q4_K_M`

原因：

- 多語能力穩定，適合即時翻譯
- 4B 版本資源需求較低
- `Q4_K_M` 檔案大約 2.5 GB，較容易在一般 Windows 主機上跑起來

如果你的顯卡與記憶體比較充足，也可以改用：

- `Qwen/Qwen3-8B-GGUF`
- 量化檔建議：`Q4_K_M` 或更高

### 在 LM Studio 下載的步驟

1. 打開 LM Studio
2. 進入 `Discover`
3. 搜尋 `Qwen/Qwen3-4B-GGUF`
4. 選擇 `Q4_K_M`
5. 按下載
6. 下載完成後，到 `Chat` 或 `Developer` 把模型載入

補充：

- LM Studio 官方文件建議，如果機器能力足夠，優先選 4-bit 或更高的量化版本。
- 如果你之後想追求更好的翻譯品質，再升到 `Qwen3-8B-GGUF`。

官方參考：

- https://lmstudio.ai/docs/app/basics/download-model
- https://lmstudio.ai/docs/developer/openai-compat
- https://huggingface.co/Qwen/Qwen3-4B-GGUF

## 第 3 步：啟用 LM Studio 本機 API

SyncTranslate 會透過 LM Studio 的 OpenAI 相容 API 溝通。

1. 在 LM Studio 載入你剛剛下載的模型
2. 啟用本機伺服器
3. 確認服務位址是 `http://127.0.0.1:1234`

SyncTranslate 預設就是連到這個位址，所以通常不用改。

## 第 4 步：安裝 SyncTranslate

### 方式 A：直接用原始碼執行

1. 安裝 Python 3.11+
2. 安裝 `uv`
3. 進到專案目錄
4. 安裝相依套件

```powershell
uv sync
```

5. 啟動程式

```powershell
uv run python .\main.py
```

### 方式 B：使用打包後版本

如果已經有 `dist\SyncTranslate\SyncTranslate.exe`，直接執行即可。

## 第 5 步：第一次啟動後的模型設定

打開 SyncTranslate 後：

1. 進到 `設定`
2. 展開 `顯示進階設定`
3. 在 `翻譯模型 LLM` 中：
   - 確認後端是 `LM Studio`
   - 按 `重新載入模型`
   - 在本地方向與遠端方向分別選擇你剛載入的模型

如果你只有載入一個模型，兩個方向都可以先選同一個。

## 第 6 步：ASR 模型說明

SyncTranslate 的語音辨識不是從 LM Studio 下載，而是走 Faster-Whisper。

預設建議：

- `large-v3`

首次使用時，Faster-Whisper 會自動下載對應權重到本機快取。

如果你的 GPU / 記憶體比較吃緊，可以改成：

- `large-v3-turbo`

它速度更快，但品質會比 `large-v3` 略低一點。

參考：

- https://huggingface.co/openai/whisper-large-v3-turbo

## 第 7 步：設定音訊裝置

1. 在 `設定 > 音訊裝置`
2. 設定：
   - `遠端輸入`
   - `遠端輸出`
   - `本地輸入`
   - `本地輸出`
3. 建議先搭配虛擬音訊線或 Voicemeeter 測試

如果你不確定怎麼接，先看：

- [音訊裝置建議配置](./音訊裝置建議配置.md)
- [直通模式排錯清單](./直通模式排錯清單.md)

## 第 8 步：第一次功能測試

1. 在 `設定` 頁按 `系統檢查`
2. 確認：
   - ASR 正常
   - LLM 正常
   - TTS 正常
3. 回到 `即時字幕`
4. 先測：
   - 遠端輸出模式切 `直通`
   - 本地輸出模式切 `翻譯`
5. 調整 `輸出音量`
6. 確認字幕、直通、TTS 都正常

## 建議的第一組實用配置

如果你想先求穩：

- LLM：`Qwen/Qwen3-4B-GGUF` + `Q4_K_M`
- ASR：`large-v3`
- 模式：
  - 遠端輸出：`直通`
  - 本地輸出：`翻譯`

如果你想先求速度：

- LLM：`Qwen/Qwen3-4B-GGUF` + `Q4_K_M`
- ASR：`large-v3-turbo`

## 常見問題

### 看得到 LM Studio，但 SyncTranslate 抓不到模型

- 確認 LM Studio 已載入模型
- 確認本機 API 已啟動
- 確認服務位址是 `http://127.0.0.1:1234`
- 回 SyncTranslate 按 `重新載入模型`

### 第一次 ASR 很慢

通常是 Faster-Whisper 正在下載模型，等第一次下載完成後就會正常。

### 只有字幕，沒有聲音

先檢查：

- `模式` 是否選對
- `輸出音量` 是否太低
- `遠端輸出 / 本地輸出` 是否選對裝置
- TTS 聲線是否可用
