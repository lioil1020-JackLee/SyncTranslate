
## 手動 smoke test 清單

1. 啟動 LM Studio，確認 local / remote model 都可用。
2. 檢查音訊路由是否正確。
3. 跑一次系統檢查，確認 ASR / LLM / TTS 都 ready。
4. 啟動即時翻譯。
5. 驗證本地方向：
   - `local_original` 有更新
   - 切 `翻譯` 時，`local_translated` 有更新且可送到 `meeting_out`
   - 切 `直通` 時，原聲可送到 `meeting_out`
6. 驗證遠端方向：
   - `meeting_original` 有更新
   - 切 `翻譯` 時，`meeting_translated` 有更新且可送到 `speaker_out`
   - 切 `直通` 時，遠端原聲可送到 `speaker_out`
7. 驗證模式與音量分離：
   - 切 `直通 / 翻譯` 不需要重調音量
   - `輸出音量` 改變後，直通與 TTS 都一起套用
   - 切成 `直通` 後，就算切頁或重新套用設定，也不會自己跳回 `翻譯`
8. 驗證 TTS 聲音切換：
   - `翻譯` 模式下可切換語音
   - 實際播放會依目前即時字幕頁選擇的語言與聲線輸出

## 目前穩定規則

- Runtime 固定為 `bidirectional`
- `remote` 與 `local` 各自有獨立的 ASR / LLM pipeline
- `passthrough` 不會停掉 ASR
- `翻譯目標` 不再承擔模式切換職責
- translated panel 在翻譯失敗時會走安全 fallback
