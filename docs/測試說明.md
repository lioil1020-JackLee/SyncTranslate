## 手動 smoke test 清單

1. 啟動 LM Studio，確認 local / remote model 都可用。
2. 檢查音訊路由是否正確。
3. 跑一次系統檢查，確認 ASR / LLM / TTS 都 ready。
4. 啟動即時翻譯。
5. 驗證本地方向：
   - `local_original` 有更新
   - 切 `翻譯` 時，`local_translated` 有更新且可送到 `meeting_out`
   - 切 `直通` 時，原聲可送到 `meeting_out`
6. 驗證遠端方向：
   - `meeting_original` 有更新
   - 切 `翻譯` 時，`meeting_translated` 有更新且可送到 `speaker_out`
   - 切 `直通` 時，遠端原聲可送到 `speaker_out`
7. 驗證模式與音量分離：
   - 切 `直通 / 翻譯` 不需要重調音量
   - `設定 > 音訊裝置` 的對應 slider 改變後，Windows 真實裝置音量應立即變化
   - 切成 `直通` 後，就算切頁或重新套用設定，也不會自己跳回 `翻譯`
8. 驗證 TTS 聲音切換：
   - `翻譯` 模式下可切換語音
   - 實際播放會依目前即時字幕頁選擇的語言與聲線輸出
9. 驗證 runtime adaptive：
   - 連續輸入短句時，ASR partial 與 LLM partial 應變得更保守，不會越跑越躁進
   - 連續輸入長句或系統延遲升高時，runtime stats 應可觀察到 adaptive mode 與實際值變化
   - 停止 session 後重新啟動，adaptive 值應回到設定檔基準，不會永久保留上一場會議的學習結果

## 目前穩定規則

- Runtime 固定為 `bidirectional`
- `remote` 與 `local` 各自有獨立的 ASR / LLM pipeline
- `passthrough` 不會停掉 ASR
- `翻譯目標` 不再承擔模式切換職責
- translated panel 在翻譯失敗時會走安全 fallback
- adaptive tuning 只在當前 session 生效，不會自動寫回設定檔
## 2026-03-30 真實裝置音量驗證

- `設定 > 音訊裝置` 的四個 slider，現在驗證的是 Windows 真實裝置音量是否變化，而不是 SyncTranslate 內部增益。
- 測試時請直接觀察 Windows 音量面板、裝置屬性，或 VoiceMeeter 對應裝置的音量變化。
- `本地輸出` 應對應本地喇叭 / 耳機的播放裝置音量。
- `遠端輸出` 應對應會議送出裝置或 VoiceMeeter 虛擬播放裝置的播放音量。
- `僅 partial 使用上下文` 開啟時，重複同一句 final ASR 應維持相同譯文；短 partial 接話才允許受前文影響。
- 補充：live caption 匯出檔需確認為「最舊在上、最新在下」，且 `[final]` 前綴已移除。
