# 測試說明

本文件對齊目前 `pyproject.toml`、`tests/` 與 `tools/`。專案測試以 pytest 為主，另有 runtime smoke 與 ASR benchmark。

## 基本檢查

```powershell
uv run python -m compileall -q app main.py tools tests
uv run pytest
```

若使用已建立的 venv，也可直接：

```powershell
.\.venv\Scripts\python.exe -m pytest
```

目前完整測試數約 371 個，實際數量會隨測試增減。

## ASR queue 與背壓測試

ASR 背壓邏輯在 `tests/test_asr_streaming_and_profiles.py`：

- `StreamingPolicy` 是否在 backlog 下 suppress partial。
- congested 狀態是否暫停 partial，避免壓力高時繼續做昂貴 partial decode。
- queue pressure final threshold 是否保守。
- backlog 高時是否提高 chunk coalescing limit。
- soft endpoint final 是否不會錯誤重啟 segment。

重點 helper：

- `_pressure_force_final_audio_ms()`
- `_drain_limit_for_backlog()`
- `SourceRuntimeV2._should_restart_after_final()`

## ASR benchmark

多樣本 benchmark：

```powershell
uv run python tools/asr_benchmark/run_multi_benchmark.py --only-lang zh --output-dir downloads/benchmark_results/manual_zh
```

英文：

```powershell
uv run python tools/asr_benchmark/run_multi_benchmark.py --only-lang en --output-dir downloads/benchmark_results/manual_en
```

單檔 streaming simulation：

```powershell
uv run python tools/asr_benchmark/streaming_sim.py --config config.yaml --audio sample.wav --reference reference.txt --source local --language zh-TW --speed 8 --output downloads/benchmark_results/manual_single
```

`streaming_sim.run_streaming_sim()` 支援：

- `asr_overrides`
- `vad_overrides`
- `streaming_overrides`
- `runtime_overrides`
- `worker_overrides`

目前可用 `worker_overrides={"queue_maxsize": 256}` 壓測正式 queue 大小，檢查 `dropped_chunks` 是否為 0。

中文 preset/model 矩陣：

```powershell
uv run python tools/asr_benchmark/run_zh_preset_matrix.py --output-dir downloads/benchmark_results/zh_preset_matrix_manual --speed 8 --queue-maxsize 256
```

這會比較 `meeting/dialogue × belle/turbo`，目前樣本包含三個故事素材、怪談 5 分鐘原音與降噪版。2026-05-07 的壓力測試中，`meeting_turbo` 平均準確率最高且 dropped chunks 為 0，因此中文預設模型改為 `large-v3-turbo`。

最近壓測基準：

- 音檔：`downloads/chinese_srt/kaidan_5min.wav`
- 語言：`zh-TW`
- source：`local`
- speed：`8x`
- queue：`256`
- 目標：`dropped_chunks = 0`

## 讀 benchmark 結果

`run_multi_benchmark.py` 會輸出每個樣本的 JSON 與 `summary.json`。常用欄位：

- `accuracy`
- `cer_normalized`
- `accuracy_dedup`
- `repetition_ratio`
- `final_count`
- `partial_count`
- `dropped_chunks`
- `speech_started_count`
- `soft_endpoint_count`
- `hard_endpoint_count`
- `streaming_params`
- `vad_params`

`dropped_chunks > 0` 表示 worker 曾丟音訊，應優先處理，因為它會造成 miss 句。

## 打包後手動測試

onedir 建立後可直接用 bundled Python 執行：

```powershell
.\dist\SyncTranslate-onedir\runtimes\shared\Scripts\python.exe .\main.py
```

若看到：

```text
llama_context: n_ctx_seq (4096) < n_ctx_train (262144)
```

這只是 llama.cpp 提示目前 context 小於模型訓練 context，不是錯誤。

若看到：

```text
ASR queue overflow (local): dropped ... chunks total
```

這是真問題，代表 ASR 已丟音訊。先檢查：

- `runtime.asr_queue_maxsize_local` 是否為 `256`。
- `runtime.asr_profile_local` 是否為 `meeting_room`。
- 是否同時開啟 local ASR、remote ASR、LLM 翻譯與 TTS。
- session report 的 `stats.asr.local.dropped_chunks` 是否持續增加。
- UI 診斷列的 `q=current/max`、`drop`、`deg`、`model`、`profile`、`enhancement` 是否符合預期。
- UI 診斷列是否顯示 `capture=possible-same-source`，若是，代表 local/remote 可能收到同一路音訊。

## 測試維護原則

新增或調整 runtime 行為時，至少補上：

- 對應 unit test。
- 若是 ASR streaming 行為，補 `test_asr_streaming_and_profiles.py`。
- 若是字幕去重/翻譯前處理，補 `test_transcript_service.py` 或 `test_audio_router_policy.py`。
- 若是 config key，補 `test_config_migration.py` / `test_settings_contract.py`。
- 若是 benchmark 工具，補 `test_asr_benchmark_smoke.py` 或以現有 smoke 覆蓋。
