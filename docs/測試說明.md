# SyncTranslate v2 測試說明

## 基本驗收

```powershell
python -m compileall -q app tests tools
uv run pytest -q
uv run python main.py --check
```

## v2 Productized Tests

```powershell
uv run pytest tests/test_productized_audio_v2.py tests/test_productized_router_v2.py -q
uv run pytest tests/test_virtual_audio_protocol_v2.py -q
uv run pytest tests/test_driver_virtual_audio_format_v2.py -q
```

這些測試覆蓋：

- app internal float32 `AudioFrame`
- ASR branch 16 kHz mono float32
- protocol v2 PCM16 stereo 48 kHz packet
- dialogue direct passthrough 不呼叫 ASR/LLM/transcript
- meeting mode 不依賴 driver/bridge

## Validation Tools

```powershell
uv run python .\tools\validation\preflight_release_check.py --mode meeting
uv run python .\tools\validation\preflight_release_check.py --mode dialogue
uv run python .\tools\validation\validate_windows_audio_runtime.py --no-capture-probe --no-bridge-probe
uv run python .\tools\validation\audio_smoke_test.py meeting-input --duration 3
uv run python .\tools\validation\audio_smoke_test.py meeting-loopback --duration 3
uv run python .\tools\validation\audio_smoke_test.py dialogue-passthrough --duration 0.1
uv run python .\tools\validation\export_diagnostics_bundle.py
```

`downloads/validation/` 是本機輸出資料夾，已在 `.gitignore` 中排除。

## Driver Scripts

本機若沒有 WDK，driver build tools 顯示 WARN 是正確行為；meeting mode 不需要 WDK。

檢查 WDK：

```powershell
powershell -ExecutionPolicy Bypass -File .\drivers\synctranslate_virtual_audio\scripts\check_wdk_environment.ps1
```

驗證已安裝 driver 與 endpoint format：

```powershell
powershell -ExecutionPolicy Bypass -File .\drivers\synctranslate_virtual_audio\scripts\verify_driver_format.ps1 -JsonOutput .\downloads\validation\driver_format.json
```

健康安裝應有：

- `SyncTranslate Virtual Audio Device` under **Sound, video and game controllers**
- `SyncTranslate Virtual Speaker` under **Audio inputs and outputs**
- `SyncTranslate Virtual Microphone` under **Audio inputs and outputs**

## Hyper-V Driver Validation

在有 WDK 的 VM 中可使用：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\hyperv\run_driver_vm_validation.ps1 -VMName win11LTSC_Test -GuestUser <user> -GuestWorkDir C:\SyncTranslate -BuildInGuest -CollectDiagnostics
```

不要提交 VM 密碼、test certificate private key 或 production signing material。

## Release Candidate 注意事項

- v2 不新增 whisper.cpp、Windows built-in ASR 或 cloud ASR。
- ASR runtime 主線是 faster-whisper/CTranslate2。
- UI language choices 不包含 `auto`。
- production release 前仍需人工驗證 production driver signing/WHQL、SmartScreen、不同 Windows 版本與常用通訊軟體。
