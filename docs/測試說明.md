## ASR queue 與背壓測試

ASR 背壓邏輯在 `tests/test_asr_streaming_and_profiles.py`：

- `StreamingPolicy` 是否在 backlog 下 suppress partial。
- congested 狀態是否暫停 partial，避免壓力高時繼續做昂貴 partial decode。
- queue pressure final threshold 是否保守。
- backlog 高時是否提高 chunk coalescing limit。
- soft endpoint final 是否不會錯誤重啟 segment。
- **final priority 狀態機**：`queue_ratio >= 0.45` 時是否進入 final priority；進入後 partial 是否被完全壓制；finalization threshold 是否減半；queue ratio 降到 0.15 後持續 8s 才退出的恢復邏輯。

重點 helper：

- `_pressure_force_final_audio_ms()`
- `_drain_limit_for_backlog()`
- `SourceRuntimeV2._should_restart_after_final()`

## ASR 優化測試

以下六個測試檔涵蓋最近 ASR 優化六個階段的新功能與信心指標：

### `tests/test_asr_auto_language_switch.py`（31 個測試）

`AutoLanguageState` dataclass 與 `observe_final_language()` 純函式的行為：

- CJK/Latin 比例分類（chinese/non_chinese/mixed）。
- 連續 streak 計數、mixed utterance 使兩方 streak 退一的邏輯。
- 冷卻期（25s）期間不觸發切換。
- 太短的 utterance（中文少於 4 CJK 字、非中文少於 3 詞）不計入 streak。
- 切換後 streak 歸零。
- manager 整合：`_observe_detected_language()` 與 `_maybe_rebuild_runtime_for_language_switch()` 的 30s 安全閥。

### `tests/test_asr_confidence_policy.py`（18 個測試）

`worker_v2.py` 的信心門檻邏輯：

- `_should_suppress_by_no_speech(result, text)`：no_speech_prob > 0.75 且 CJK 數為 0 且文字過短時壓制；中文文字或長段不應被壓制。
- `_has_compression_loop_risk(result)`：compression_ratio > 2.6 時回傳 `True`；正常範圍內回傳 `False`。

### `tests/test_faster_whisper_metadata.py`（21 個測試）

`faster_whisper_adapter.py` 的 segment 元資料：

- `SegmentMetadata` 欄位（text, start, end, avg_logprob, no_speech_prob, compression_ratio）。
- `TranscribeResult` 擴充欄位（segments, avg_logprob, max_no_speech_prob, max_compression_ratio）。
- `_collect_segment_metadata()`：duration-weighted avg_logprob 聚合；max 聚合（no_speech_prob、compression_ratio）；空 segment list 的邊界情況。

### `tests/test_display_punctuation.py`（22 個測試）

`apply_display_punctuation()` 的標點規則：

- 短文字（CJK < 4）不補標點。
- 含問詞結尾補 `？`。
- 長 CJK 段（>= 8）補 `。`。
- 已有句末標點時不疊加。
- `is_final=False` 原樣透傳。
- `enabled=False` 原樣透傳。

### `tests/test_asr_regression_manifest.py`（20 個測試）

`run_regression_corpus.py` 的 manifest 解析：

- `load_manifest()`：YAML 載入與結構驗證。
- `parse_samples()`：欄位映射（lang, mode, audio, reference, thresholds）。
- `validate_sample_paths()`：缺失音檔與 reference 的錯誤回報；`--skip-missing` 旗標略過缺失路徑。

### `tests/test_asr_regression_runner.py`（25 個測試）

`run_regression_corpus.py` 的執行與報告邏輯：

- `check_thresholds(result, thresholds)`：accuracy/cer/dropped_chunks 門檻比較。
- `aggregate_summary(results)`：跨樣本統計彙總。
- `write_summary_json(summary, path)`：JSON 輸出格式與欄位完整性。
- `write_summary_md(summary, path)`：Markdown report 格式。

### `tests/test_asr_local_benchmark_corpus.py`（3 個測試）

`prepare_local_benchmark_corpus.py` 的本機素材切段與門檻校準：

- `parse_subtitles()`：支援 SRT/VTT 時間軸、BOM 與簡單字幕標籤清理。
- `_group_cues()`：依目標秒數與最大秒數將字幕 cue 組成 regression sample。
- `_thresholds_for()`：針對 `subtitle_aligned` reference 使用較適合的 accuracy/missing sentence 門檻。

## ASR regression corpus

manifest 驅動的回歸測試，涵蓋 zh/en/mixed × 多場景樣本。優先使用 curated stable baseline corpus：保留雙模式穩定通過的本機 benchmark 段落，並用已實測穩定的 FLEURS/ASCEND 樣本補足 20/10/5；公開資料集 corpus 保留為 fallback 與外部壓力測試。

建立本機 benchmark corpus：

```powershell
uv run --with datasets==2.18.0 --with librosa --with soundfile --with numpy python tools/asr_benchmark/prepare_local_benchmark_corpus.py --force
```

目前工具會建立 35 筆樣本：

- 中文：20 筆。
- 英文：10 筆。
- 中英混合：5 筆。
- 類型包含 `news`、`novel`、`short_video`、`mixed_dialogue`。
- reference quality 包含 `subtitle_aligned` 與 `dataset_transcript`。此 corpus 是穩定 baseline gate；被移出的高難度短影音、故事尾段與快語速混語樣本應放在 stress 測試觀察，不拿來卡 baseline。

跑 baseline：

```powershell
uv run python tools/asr_benchmark/run_regression_corpus.py --manifest downloads/asr_regression_local/manifest.yaml --config config.yaml --models turbo --modes meeting,dialogue --speed 16 --quick --output-dir downloads/benchmark_results/asr_regression_local_baseline
```

建立公開資料集 fallback corpus：

```powershell
uv run --with datasets==2.18.0 --with librosa --with soundfile --with numpy python tools/asr_benchmark/prepare_regression_corpus.py --force
uv run python tools/asr_benchmark/run_regression_corpus.py --manifest downloads/asr_regression/manifest.yaml --config config.yaml --models turbo --modes meeting,dialogue --speed 16 --quick --output-dir downloads/benchmark_results/asr_regression_baseline
```

其他常用旗標：

- `--models turbo,belle`：指定要測試的模型清單；runner 會映射到實際 runtime 模型。
- `--modes meeting,dialogue`：指定 endpoint profile 清單。
- `--speed 8`：音訊回放倍速。
- `--skip-missing`：略過 manifest 中路徑不存在的樣本，不中止執行。

2026-05-11 local benchmark corpus baseline 已跑：

- `downloads/benchmark_results/asr_regression_local_20260511_meeting/summary.md`
- `downloads/benchmark_results/asr_regression_local_20260511_dialogue/summary.md`

兩份 baseline 都是 35 筆樣本、0 個 runtime error。meeting 目前是 35/35 passed，dialogue 目前是 35/35 passed。runner 的 `summary.json` 會保存每筆 transcript，方便日後追查退步樣本。

## 讀 benchmark 結果

`run_multi_benchmark.py` 會輸出每個樣本的 JSON 與 `summary.json`。常用欄位：

- `accuracy`
- `cer_normalized`
- `accuracy_dedup`
- `repetition_ratio`
- `final_count`
- `partial_count`
- `dropped_chunks`
- `speech_started_count`
- `soft_endpoint_count`
- `hard_endpoint_count`
- `streaming_params`
- `vad_params`

`dropped_chunks > 0` 表示 worker 曾丟音訊，應優先處理，因為它會造成 miss 句。

## 打包後手動測試

onedir 建立後可直接用 bundled Python 執行：

```powershell
.\dist\SyncTranslate-onedir\runtimes\shared\Scripts\python.exe .\main.py
```

若看到：

```text
llama_context: n_ctx_seq (4096) < n_ctx_train (262144)
```

這只是 llama.cpp 提示目前 context 小於模型訓練 context，不是錯誤。

若看到：

```text
ASR queue overflow (local): dropped ... chunks total
```

這是真問題，代表 ASR 已丟音訊。先檢查：

- `runtime.asr_queue_maxsize_local` 是否為 `256`。
- `runtime.asr_profile_local` 是否為 `meeting_room`。
- 是否同時開啟 local ASR、remote ASR、LLM 翻譯與 TTS。
- session report 的 `stats.asr.local.dropped_chunks` 是否持續增加。
- UI 診斷列的 `q=current/max`、`drop`、`deg`、`model`、`profile`、`enhancement` 是否符合預期。
- UI 診斷列是否顯示 `fp=on`（final priority 已啟用）或 `lang=auto→zh-TW`（auto 語言切換已生效）。
- UI 診斷列是否顯示 `capture=possible-same-source`，若是，代表 local/remote 可能收到同一路音訊。

## 測試維護原則

新增或調整 runtime 行為時，至少補上：

- 對應 unit test。
- 若是 ASR streaming 行為，補 `test_asr_streaming_and_profiles.py`。
- 若是 ASR 信心指標或 segment metadata，補 `test_asr_confidence_policy.py` 或 `test_faster_whisper_metadata.py`。
- 若是 auto 語言切換，補 `test_asr_auto_language_switch.py`。
- 若是字幕去重/翻譯前處理，補 `test_transcript_service.py` 或 `test_audio_router_policy.py`。
- 若是 config key，補 `test_config_migration.py` / `test_settings_contract.py`。
- 若是 benchmark 工具，補 `test_asr_benchmark_smoke.py`、`test_asr_regression_manifest.py` 或 `test_asr_regression_runner.py`。
- 若是 UI 顯示層後處理，補 `test_display_punctuation.py`。
