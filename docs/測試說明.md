# 測試說明

本專案的測試分成三層：快速靜態/編譯檢查、pytest 單元與可控整合測試、實機 runtime smoke。一般程式碼修改至少應跑 compileall 與相關 pytest。

## 快速檢查

```powershell
uv run python -m compileall -q app main.py tools tests
```

這會確認 Python 檔案都能編譯，適合在重構或刪檔後先跑。

## 全量 pytest

```powershell
uv run pytest
```

測試路徑由 `pyproject.toml` 設定為 `tests/`，並把專案根目錄加入 `pythonpath`。

## 常用子集

ASR v2：

```powershell
uv run pytest tests/test_asr_v2_endpointing.py tests/test_asr_streaming_and_profiles.py tests/test_asr_frontend_v2.py tests/test_asr_enhancement_v2.py
```

翻譯：

```powershell
uv run pytest tests/test_translation_stitcher.py tests/test_translation_manager_profiles.py tests/test_translation_provider_local_llama.py
```

音訊與路由：

```powershell
uv run pytest tests/test_audio_router_core.py tests/test_audio_router_policy.py tests/test_audio_routing_virtual_devices.py
```

UI / 設定：

```powershell

```

## 測試分布

主要測試檔：

- `test_app_factory.py`：CLI 與 app factory。
- `test_external_runtime.py`：外部 runtime path 與環境變數。
- `test_config_migration.py` / `test_settings_contract.py`：設定遷移與 UI 設定契約。
- `test_audio_router_core.py` / `test_audio_router_policy.py`：AudioRouter 行為。
- `test_asr_*`：ASR v2、語言 profile、frontend、endpointing、postprocess、resampling。
- `test_hallucination_filter.py`：ASR 幻覺與 overlay 過濾。
- `test_translation_*`：本地翻譯 provider、manager、stitcher。
- `test_tts_style_policy.py`：TTS voice/style policy。
- `test_transcript_*`：字幕 buffer 與後處理。
- `test_healthcheck_service.py` / `test_diagnostics_export.py`：健康檢查與診斷匯出。
- `test_asr_benchmark_smoke.py`：benchmark 工具的 CER/WER 與報表 smoke。

已移除只覆蓋舊 compatibility prototype 的測試，例如舊 ASR stream worker、舊 audio pipeline stage package、未接入 UI controller prototype。

## Runtime Smoke

pytest 盡量避免依賴實體裝置、CUDA、GGUF 模型與 TTS 網路服務。需要驗證真實 runtime 時使用：

```powershell
uv run python .\tools\runtime_smoke\run_runtime_smoke.py --config config.yaml
```

包含 pytest 與 streaming replay：

```powershell
uv run python .\tools\runtime_smoke\run_runtime_smoke.py --config config.yaml --with-pytest --with-streaming-replay all
```

## ASR Benchmark

單檔 benchmark：

```powershell
uv run python tools/asr_benchmark/run_benchmark.py --audio sample.wav --reference reference.txt --profile default --output results.jsonl
```

報表：

```powershell
uv run python tools/asr_benchmark/report.py results.jsonl
```

串流模擬：

```powershell
uv run python tools/asr_benchmark/streaming_sim.py --config config.yaml --audio sample.wav --reference reference.txt --source remote --language en --speed 8 --output downloads/benchmark_results/streaming
```

## 測試撰寫原則

- 新 runtime 行為優先測現行模組，不為舊 shim 新增測試。
- 涉及音訊、LLM、TTS 的測試要 mock 外部依賴。
- 重構後刪除只驗證舊檔案存在的測試。
- 若功能需要實體裝置或模型，放到 runtime smoke 或 benchmark，不塞進一般 pytest。
