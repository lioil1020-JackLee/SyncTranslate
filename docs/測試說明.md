# 測試說明

## 安裝

完整本地測試環境：

```powershell
uv sync --locked --extra local
```

如果只需要一般 Python 測試，不需要額外本地 ASR 依賴，也可以先用：

```powershell
uv sync --locked
```

## 執行方式

### 全部 `unittest`

```powershell
uv run python -m unittest discover -s tests -p "test_*.py" -v
```

### 單跑某幾組

```powershell
uv run python -m unittest tests.test_ui_behavior tests.test_asr_backend_routing tests.test_asr_factory tests.test_asr_v2_endpointing
```

### 系統檢查

```powershell
uv run python .\main.py --check
```

## 測試目的

目前測試主要覆蓋：

- ASR 路由規則是否正確
- endpointing 與 runtime stats 是否正確
- UI 與設定檔是否對齊
- migration 後 config 是否仍能正確載入
- transcript / translation / TTS pipeline 是否基本可用

## 建議驗收流程

自動測試之外，這個專案仍建議搭配手動驗收：

1. 跑 `--check`
2. 開啟程式確認 UI 能正常載入
3. 用中文單通道測試
4. 用雙通道同時發話測試
5. 匯出 `logs/session_reports/*.json`
6. 檢查：
   - `resolved_backend`
   - `device_effective`
   - `speech_started_count`
   - `partial_count`
   - `final_count`
   - `dropped_chunks`

這些欄位通常比單看字幕內容更容易定位瓶頸。
