# 設定說明

SyncTranslate 的設定以 YAML 儲存，主要檔案是 `config.yaml`，範本是 `config.example.yaml`。啟動時會載入 YAML，遷移舊欄位，最後轉成 `AppConfig`。

## 載入規則

1. CLI 指定 `--config` 時使用該檔案。
2. 未指定時，預設使用程式目錄或目前工作目錄的 `config.yaml`。
3. 如果 `config.yaml` 不存在，會讀取 `config.example.yaml`。
4. 如果兩者都不存在，會用 `AppConfig` 預設值建立設定檔。

設定載入與儲存邏輯在 `app/infra/config/settings_store.py`。

## 主要區塊

### runtime

控制目前 session 的 runtime 行為。

常用欄位：

- `local_asr_language`
- `remote_asr_language`
- `local_tts_voice`
- `remote_tts_voice`
- `asr_profile_local`
- `asr_profile_remote`
- `asr_queue_maxsize_local`
- `asr_queue_maxsize_remote`
- `tts_accept_stable_partial`
- `tts_partial_min_chars`

`local_asr_language` 與 `remote_asr_language` 可以使用 `auto`、`zh-TW`、`en`、`ja`、`ko`、`th` 等語言碼。`auto` 會讓 faster-whisper 自動偵測語言。

### language

控制字幕翻譯方向。

- `local_source`：本機麥克風原文語言。
- `local_target`：本機語音要翻成的目標語言。
- `meeting_source`：遠端會議原文語言。
- `meeting_target`：遠端語音要翻成的目標語言。

UI 的即時字幕頁會同步更新這些欄位。

### audio

控制輸入、輸出與音量。

常用欄位：

- `local_input_device`
- `remote_input_device`
- `local_output_device`
- `remote_output_device`
- `local_input_gain`
- `remote_input_gain`
- `local_output_volume`
- `remote_output_volume`

裝置 selector 由 `app/infra/audio/device_registry.py` 解析。Windows endpoint volume 由 `device_volume_controller.py` 透過 PowerShell script 控制。

### asr_profiles

定義 ASR profile。目前 runtime 會依語言 family 選擇實際 profile：

- 中文語言走中文 ASR profile。
- 非中文或 auto 走非中文 faster-whisper profile。
- disabled/none 會停用該來源 ASR。

每個 profile 可包含：

- `model`
- `device`
- `compute_type`
- `endpoint_profile`
- `vad`
- `streaming`
- `frontend`
- `asr`

語言細部調整集中在 `app/infra/asr/language_profiles.py`。

### llm_profiles

定義本地翻譯模型。現在的主要 backend 是 in-process llama.cpp。

重要欄位：

- `backend`
- `model`
- `model_path`
- `context_size`
- `gpu_layers`
- `threads`
- `temperature`
- `timeout_s`

預設模型位置通常是：

```text
runtimes/models/llm/hy-mt1.5-7b.gguf
```

### tts

控制 TTS provider、voice、語速、音量與 style。Edge TTS voice fallback 在 `app/infra/tts/voice_policy.py`。

常見 voice：

- 中文：`zh-TW-HsiaoChenNeural`
- 英文：`en-US-JennyNeural`
- 日文：`ja-JP-NanamiNeural`
- 韓文：`ko-KR-SunHiNeural`
- 泰文：`th-TH-PremwadeeNeural`

## 輸出模式

每個 channel 可能是：

- `tts`：翻譯後合成語音並播放。
- `subtitle_only`：只顯示字幕。
- `passthrough`：不翻譯或不 TTS，直接把音訊送到輸出端。

實際判斷在 UI 與 `TTSManager` 間同步。一般規則是：ASR 語言為 none 時偏向只顯示/停用；翻譯目標為 none 時偏向 passthrough；TTS voice 有效時使用 tts。

## 設定遷移

`_config_migration.py` 會處理舊欄位，例如：

- 舊 `session_mode`。
- 舊 `openai` 設定。
- 舊 ASR engine 名稱。
- 舊 VAD backend 名稱。
- 舊 shared profile 欄位。

遷移後會轉成目前的 `AppConfig` 結構。

## 建議維護方式

- 修改 schema 時同步更新 `config.example.yaml` 與本文件。
- 新增 UI 設定時確認 `LocalAiPage.update_config()` 與 `apply_config()` 都有涵蓋。
- 新增外部 YAML key 時同步更新 `_schema_parser.py` 與 `_config_serialization.py`。
- 不再新增只為舊欄位存在的 shim；必要時放在 migration 中一次性轉換。
