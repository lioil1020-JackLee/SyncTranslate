# 設定說明

SyncTranslate 的主要設定檔是 `config.yaml`，範本是 `config.example.yaml`。設定會由 `app/infra/config/settings_store.py` 載入，經過 migration 後轉成 `app/infra/config/schema.py` 的 `AppConfig`。

**標準配置**：使用 `routing_mode: synctranslate_virtual_audio`，依賴 SyncTranslate Audio Driver 提供虛擬音訊設備。不再推薦 Voicemeeter 或其他虛擬音訊軟體。

## 載入順序

1. CLI 可用 `--config` 指定設定檔。
2. 未指定時讀取專案根目錄的 `config.yaml`。
3. 若沒有 `config.yaml`，會使用 `config.example.yaml`。
4. **舊版 key 會由 `_config_migration.py` 自動轉成新版 schema**。

## audio

`AudioRouteConfig` 對應 key。**建議使用 `synctranslate_virtual_audio` 模式（標準做法，無需外部虛擬音訊工具）**。

```yaml
audio:
  meeting_in: ""              # 遠端/會議音訊輸入（由虛擬喇叭自動識別）
  microphone_in: ""           # 本地麥克風輸入（由虛擬麥克風自動識別）
  speaker_out: ""             # 本地喇叭輸出（自動路由）
  meeting_out: ""             # 送回會議的輸出（虛擬麥克風內部路由）
  routing_mode: synctranslate_virtual_audio  # 新標準模式
  virtual_audio:
    speaker_name: SyncTranslate Virtual Speaker
    microphone_name: SyncTranslate Virtual Microphone
    bridge_enabled: true
    bridge_path: runtimes\audio\sync_audio_bridge.exe
    sample_rate: 48000
    frame_ms: 10
    target_latency_ms: 120
    require_driver: true      # 需要先安裝驅動
  system_devices:
    capture_role: communications
    render_role: communications
    follow_system_default: true
    fallback_to_multimedia_role: true
    exclude_virtual_devices: true
  call_translation:
    listen_remote_original: true
    listen_remote_translation: true
    output_local_original: false
    output_local_translation: true
```

**模式選擇**：

| 模式 | 說明 | 前置條件 | 建議用途 |
|-----|------|--------|--------|
| `synctranslate_virtual_audio` | **標準模式** — 透過虛擬喇叭/麥克風接入音訊 | ✅ 已安裝 SyncTranslate Audio Driver；自用版需 Test Mode | 個人自用、內測、願意開 Test Mode 的 GitHub 朋友 |

`synctranslate_virtual_audio` 是唯一產品路由。舊設定中的其他模式值會被正規化到此模式；若 driver 或 bridge 不可用，App 會顯示診斷與阻擋原因，不會引導使用第三方虛擬音訊軟體。

**key 語意**：

- `routing_mode`：路由模式選擇
- `meeting_in`：遠端/會議音訊輸入
- `microphone_in`：本地音訊輸入
- `speaker_out`：本地喇叭輸出
- `meeting_out`：送回會議端的輸出
- `virtual_audio`：SyncTranslate 虛擬喇叭、虛擬麥克風與 bridge 參數
- `system_devices`：真實麥克風/耳機的系統預設角色解析策略
- `call_translation`：通話翻譯輸出策略，由 UI 設定與 TTS 狀態套用

**舊音訊設定升級**：

若使用者之前使用第三方虛擬音訊工具，升級到新版本時：
1. App 會自動檢測舊設定並執行 migration
2. `routing_mode` 會自動設為 `synctranslate_virtual_audio`
3. 用戶需要按照「驅動安裝」流程安裝 SyncTranslate Audio Driver 一次
4. 之後無需再手動設定，App 會自動用虛擬設備

## language

```yaml
language:
  meeting_source: en
  meeting_target: zh-TW
  local_source: zh-TW
  local_target: en
```

這是 UI 與預設語言方向。實際 ASR 與翻譯可被 `runtime.remote_asr_language`、`runtime.local_asr_language`、`runtime.remote_translation_target`、`runtime.local_translation_target` 覆寫。

## runtime 常用設定

### ASR pipeline

```yaml
runtime:
  asr_pipeline: v2
  asr_v2_backend: faster_whisper_v2
  asr_v2_endpointing: neural_endpoint
```

目前 `asr_pipeline` 會正規化成 `v2`。`neural_endpoint` 會依 VAD 設定解析成 `silero_vad`。

### 音訊流

```yaml
runtime:
  sample_rate: 48000
  chunk_ms: 40
```

`chunk_ms` 越小延遲越低，但 ASR worker 壓力越高。現行預設 `40ms`。

### ASR 語言

```yaml
runtime:
  remote_asr_language: auto
  local_asr_language: zh-TW
```

支援值：

- `auto` 或空字串：faster-whisper 自動語言。
- `zh-TW` / `zh-CN` / `zh` / `cmn` / `yue`：Chinese family。
- `en`、`ja`、`ko`、`th` 等：非中文 family。
- `none`：停用該 channel ASR。

目前模型路由：

- Chinese family 使用 `asr_channels.local`，預設 `large-v3-turbo`；`belle-zh-ct2` 保留為設定 UI 的 ASR「模型」備選。
- 非中文與 `auto` 使用 `asr_channels.remote`，預設 `large-v3-turbo`。

**`auto` 模式動態 profile 切換**：設為 `auto` 時，ASR manager 會追蹤每個 final utterance 的語言家族（依 CJK/Latin 字元比例分類）。連續 3 個 final 屬於同一家族（Chinese 或非 Chinese），且距上次切換超過 25s 冷卻期，effective profile 即自動切換。runtime 重建另有 30s 安全閥防止短時間內反覆重建。切換狀態可在 `stats.asr.<source>.auto_language` 查看，UI 診斷列會顯示 `lang=auto→zh-TW`（切換進行中時為 `(pending)`）。

### ASR profile

```yaml
runtime:
  asr_profile_local: meeting_room
  asr_profile_remote: meeting_room
```

可用 profile 定義在 `app/infra/asr/endpoint_profiles.py`：

- `default`
- `meeting_room`
- `headset`
- `noisy_environment`
- `max_accuracy`
- `low_latency`
- `turn_taking`

建議一般字幕使用 `meeting_room`。`turn_taking` 延遲較低，但更容易增加 decode 壓力；設定 UI 會把 ASR queue 自動維持在至少 `256`，並由 worker 在壓力升高時降低 partial decode 頻率或暫停 partial。

### ASR queue

```yaml
runtime:
  asr_queue_maxsize_local: 256
  asr_queue_maxsize_remote: 256
```

queue 太小會在 final decode 或 GPU 忙碌時爆掉，log 會出現：

```text
ASR queue overflow (local): dropped ... chunks total
```

這代表音訊 chunk 已被丟棄，會造成漏句。現行預設與 UI 防呆值都是 `256`。若 queue 接近高水位，worker 會先暫停 partial、拉長 partial 間隔並合併更多 chunk；若仍持續遞增，應降低 ASR/LLM/TTS 負載，而不是無限制增加 queue，否則延遲會變長。

### ASR final priority

當 ASR queue 壓力升高或 final 輸出延遲時，`streaming_policy.py` 的 final priority 狀態機會自動進入壓制 partial 模式，保護 final 輸出品質。預設已啟用，進入/退出門檻可透過以下設定調整：

```yaml
runtime:
  asr_final_priority_enabled: true
  asr_final_priority_queue_ratio: 0.45
  asr_final_priority_latency_ms: 1800
  asr_final_priority_recover_queue_ratio: 0.15
  asr_final_priority_recover_after_ms: 8000
```

各 key 說明：

- `asr_final_priority_enabled`：全域開關，設為 `false` 可完全停用此狀態機。預設 `true`。
- `asr_final_priority_queue_ratio`：queue 使用比例達到此值（0.0–1.0）時進入 final priority。預設 `0.45`（queue 45% 滿）。
- `asr_final_priority_latency_ms`：單次 final 延遲超過此毫秒數時觸發進入。預設 `1800`ms。
- `asr_final_priority_recover_queue_ratio`：queue 使用比例降到此值以下時開始計算恢復倒數。預設 `0.15`。
- `asr_final_priority_recover_after_ms`：queue ratio 低於恢復門檻後，需持續此毫秒數才正式退出 final priority 模式。預設 `8000`ms（8s）。

進入 final priority 時 `StreamingDecision.final_priority_active = True`，`final_priority_reason` 說明觸發原因；UI 診斷列顯示 `fp=on`。

### ASR 顯示用標點

ASR 辨識結果通常不含句末標點。啟用此功能後，`apply_display_punctuation()` 會在 UI 渲染層對中文 final 補全輕量標點，不影響翻譯管線。預設關閉。

```yaml
runtime:
  asr_display_punctuation_enabled: false
  asr_display_punctuation_languages: "zh"
  asr_display_punctuation_mode: "lightweight"
```

各 key 說明：

- `asr_display_punctuation_enabled`：全域開關。預設 `false`。
- `asr_display_punctuation_languages`：套用標點的目標語言（逗號分隔）。預設 `"zh"`（中文）。目前實作只對 CJK 字元比例高的 final 動作，非中文 final 自然略過。
- `asr_display_punctuation_mode`：標點規則模式。目前只有 `"lightweight"`，規則保守：CJK < 4 字的片段略過；含問詞結尾補 `？`；CJK >= 8 字的長段補 `。`；已有句末標點時不疊加。

### ASR frontend

```yaml
runtime:
  asr_frontend_enabled: true
  asr_frontend_target_rms: 0.06
  asr_frontend_max_gain: 3.5
  asr_frontend_highpass_alpha: 0.96
  asr_enhancement_enabled: true
  asr_enhancement_noise_reduce_strength: 0.28
  asr_enhancement_noise_adapt_rate: 0.18
  asr_enhancement_music_suppress_strength: 0.12
```

注意：`runtime.asr_enhancement_enabled` 是全域開關，但 `language_profiles.py` 會讓中文 profile 停用頻譜增強器。也就是：

- 中文 belle：AGC/high-pass 保留，頻譜增強關閉。
- 非中文/auto：依 runtime 設定啟用頻譜增強。

### partial/final 與顯示

```yaml
runtime:
  display_partial_strategy: stable_only
  stable_partial_min_repeats: 3
  partial_stability_max_delta_chars: 6
  early_final_enabled: false
  enable_postprocessor: true
  enable_partial_stabilization: true
```

UI 原文 ASR panel 會顯示 `[partial]` / `[final]` 狀態。翻譯 panel 不顯示 `[final]`，因為翻譯只在 ASR final 後送出。

### 翻譯

```yaml
runtime:
  remote_translation_enabled: true
  local_translation_enabled: true
  remote_translation_target: zh-TW
  local_translation_target: en
  llm_queue_maxsize_local: 24
  llm_queue_maxsize_remote: 24
  llm_streaming_tokens: 16
  max_pipeline_latency_ms: 3200
```

翻譯 queue 由 `app/application/translation_dispatcher.py` 管理。`max_pipeline_latency_ms` 過低會讓過期 ASR event 被丟棄，過高則可能增加字幕延遲。

### TTS

```yaml
runtime:
  tts_output_mode: passthrough
  remote_tts_voice: none
  local_tts_voice: none
  remote_tts_enabled: false
  local_tts_enabled: false
```

`tts_output_mode` 可用：

- `tts`
- `subtitle_only`
- `passthrough`

`none` voice 表示不啟用 TTS，只顯示字幕或 passthrough。

## asr_channels

`asr_channels.local` 與 `asr_channels.remote` 是實際 ASR profile slot。

`AsrConfig` 重要欄位：

- `engine`
- `model`
- `device`
- `compute_type`
- `beam_size`
- `final_beam_size`
- `condition_on_previous_text`
- `final_condition_on_previous_text`
- `initial_prompt`
- `hotwords`
- `temperature_fallback`
- `no_speech_threshold`
- `hallucination_filter`
- `vad`
- `streaming`

中文 profile 會由 `language_profiles.py` 在 runtime 再覆寫部分欄位，例如 `no_speech_threshold=0.25`、Silero threshold `0.45`、endpoint profile `meeting_room`。

## ASR v3 準確度設定

ASR v3 以 `runtime.asr_accuracy_mode` 控制延遲與準確率取捨：

- `low_latency`：關閉 final rescue 與中文 fallback，partial/final 優先低延遲。
- `balanced`：預設模式。只有低信心 final 會進入 rescue。
- `high_accuracy`：低信心 final 會更積極嘗試同模型 rescue 與中文 fallback。

相關 runtime key：

```yaml
runtime:
  asr_accuracy_mode: balanced
  asr_final_rescue_enabled: true
  asr_final_rescue_max_attempts: 1
  asr_final_rescue_min_avg_logprob: -1.0
  asr_final_rescue_max_no_speech_prob: 0.65
  asr_final_rescue_max_compression_ratio: 2.4
  asr_final_rescue_min_chars: 4
  asr_chinese_fallback_enabled: true
  asr_chinese_fallback_model: .\runtimes\models\belle-zh-ct2
```

`asr_final_rescue_*` 只影響 final 低信心判斷，不會讓每個 partial 都重跑。`asr_chinese_fallback_model` 只在中文 final 低信心、queue 壓力未過高且模式不是 `low_latency` 時使用。
