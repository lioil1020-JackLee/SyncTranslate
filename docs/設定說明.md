# 設定說明

## 設定檔位置

- 主設定檔：`config.yaml`
- 範例設定檔：`config.example.yaml`
- 結構定義：[`app/infra/config/schema.py`](/e:/py/SyncTranslate/app/infra/config/schema.py)
- 載入與儲存：[`app/infra/config/settings_store.py`](/e:/py/SyncTranslate/app/infra/config/settings_store.py)

## 目前設定策略

目前 UI 已移除舊版快速預設切換，改成一組固定的穩定預設。若要細調，可在設定頁勾選 `顯示進階設定`。

內建預設重點：

- ASR：`large-v3`
- 執行取樣率：`48000`
- chunk：`40ms`
- pre-roll：`220ms`
- 內部播放增益：固定 `100%`
- 字幕 profile：`live_caption_fast`
- 語音 profile：`speech_output_natural`

## 重要區塊

### `audio`

四個音訊路由欄位：

- `meeting_in`
- `microphone_in`
- `speaker_out`
- `meeting_out`

### `asr` / `asr_channels`

- `asr`：共用 ASR 預設
- `asr_channels.local`：本地方向
- `asr_channels.remote`：遠端方向

常用欄位：

- `model`
- `device`
- `compute_type`
- `beam_size`
- `vad.*`
- `streaming.*`

### `llm` / `llm_channels`

- `llm`：共用 LLM 預設
- `llm_channels.local`：中文轉英文方向
- `llm_channels.remote`：英文轉中文方向

常用欄位：

- `backend`
- `base_url`
- `model`
- `temperature`
- `top_p`
- `max_output_tokens`
- `repeat_penalty`
- `caption_profile`
- `speech_profile`

### `tts` / `meeting_tts` / `local_tts`

目前主要使用 `edge_tts`。

- `meeting_tts`
  - 給遠端翻譯輸出使用
- `local_tts`
  - 給本地翻譯輸出使用
- `tts_channels.local` / `tts_channels.remote`
  - 每個輸出通道的覆寫值

### `runtime`

目前較常調整的欄位：

- `sample_rate`
- `chunk_ms`
- `passthrough_gain`
- `tts_gain`
- `latency_mode`
- `asr_pre_roll_ms`
- `display_partial_strategy`
- `stable_partial_min_repeats`
- `partial_stability_max_delta_chars`
- `asr_partial_min_audio_ms`
- `asr_partial_interval_floor_ms`
- `llm_partial_interval_floor_ms`
- `early_final_enabled`
- `tts_accept_stable_partial`
- `tts_partial_min_chars`
- `tts_use_speech_profile`
- `local_echo_guard_enabled`
- `local_echo_guard_resume_delay_ms`
- `remote_echo_guard_resume_delay_ms`
- `tts_output_mode`

目前這些欄位已可在 `設定 > 顯示進階設定` 內調整。

## 輸出模式

每個方向現在都由即時字幕頁的 `模式` 控制：

- `直通`
  - 該方向走 `passthrough`
  - 不做翻譯語音輸出
- `翻譯`
  - 該方向走 `tts`
  - `翻譯目標` 與 `TTS 語音` 會生效

補充：

- `翻譯目標` 已不再提供 `無`
- 是否直通不再靠把目標語言設成 `無` 來切換
- 真實裝置音量改由 `設定 > 音訊裝置` 的四個百分比 slider 控制

## 設定遷移

`settings_store.py` 會在讀取舊設定時做整理，例如：

- 舊版翻譯目標欄位對應到現在的 target 欄位
- 舊版 TTS 欄位對應到 `meeting_tts` / `local_tts`
- 方向專用 queue 欄位補齊
- 非法 `tts_output_mode` 回退到合法值

## Runtime 產物

執行期間會在運行目錄下留下診斷資料，例如：

- runtime event log
- system check 輸出
- crash log
## 2026-03-30 真實裝置音量

- `設定 > 音訊裝置` 的四個音量控制，現在直接對應真實裝置音量，不再是程式內部增益。
- `遠端輸入`、`本地輸入` 對應 Windows 錄音裝置 endpoint volume。
- `遠端輸出`、`本地輸出` 對應 Windows 播放裝置 endpoint volume。
- VoiceMeeter 虛擬裝置若被選為音訊裝置，也會直接控制該裝置的 Windows endpoint volume。
- UI 顯示為 `0% ~ 100%` 百分比。
- SyncTranslate 內部 `passthrough_gain` / `tts_gain` 現在固定維持 `100%`，不再承擔裝置音量功能。
- `僅 partial 使用上下文` 開啟時，只讓 partial 翻譯參考前文；final 翻譯仍只看當句。
