# SyncTranslate 快速安裝手冊（從零開始）

本手冊是給 **完全沒有相關經驗** 的使用者使用。  
目標是從一台剛準備好要安裝的 Windows 電腦開始，逐步完成以下項目：

1. 安裝 **LM Studio**
2. 在 LM Studio 下載並載入 **hy-mt1.5-7b（Tencent，Q4_K_M）**
3. 安裝 **VoiceMeeter**
4. 安裝 **SyncTranslate 打包後的 onedir 版本**
5. 啟動並完成第一次設定與測試

---

## 先說明：哪些會自動、哪些不會

本安裝流程分成兩種：

### 會自動處理的部分
安裝腳本會協助你：
- 建立安裝資料夾
- 整理安裝檔到指定資料夾
- 啟動 LM Studio 安裝程式
- 啟動 VoiceMeeter 安裝程式
- 解壓或複製 SyncTranslate onedir

### 仍需要你手動操作的部分
因為這些步驟通常需要使用者在畫面上點選、登入、同意或確認，所以不建議假裝成完全自動：
- LM Studio 首次啟動
- 在 LM Studio 搜尋與下載模型
- 在 LM Studio 載入模型
- 在 LM Studio 開啟 Local Server
- VoiceMeeter 的音訊路由設定
- SyncTranslate 內的模型與音訊裝置設定

也就是說，這份手冊是 **半自動安裝**。  
你只要照著做，大部分繁瑣步驟都會簡化，但仍有少數畫面需要你手動操作。

---

## 一、安裝前請先確認

### 1. 你的電腦需求
請先確認電腦至少符合以下條件：

- 作業系統：Windows 10 或 Windows 11（64 位元）
- CPU：支援 AVX2
- 記憶體：建議至少 16 GB RAM
- 顯示卡：建議至少 4 GB 獨立 VRAM
- 網路：需要可以上網下載安裝檔與模型
- 權限：建議你有 **系統管理員權限**

### 2. 你需要準備的檔案
請先準備以下三個檔案，並放在和安裝腳本相同的資料夾：

- `LM-Studio-Setup.exe`
- `VoiceMeeterSetup.zip` 或官方下載的 `VoicemeeterSetup_v2122.zip`
- `SyncTranslate-onedir.zip` 或 GitHub Release 產出的 `SyncTranslate-onedir-windows.zip`

安裝腳本會直接使用這三個檔案，不需要你先手動修改 `.ps1` 內容。

下載來源：

- `LM-Studio-Setup.exe`
  下載頁：<https://lmstudio.ai/download>
- `VoiceMeeterSetup.zip` / `VoicemeeterSetup_v2122.zip`
  官方頁：<https://vb-audio.com/Voicemeeter/banana.htm>
- `SyncTranslate-onedir-windows.zip`
  發佈頁：<https://github.com/lioil1020-JackLee/SyncTranslate/releases>

說明：

- LM Studio 的實際下載檔名可能會隨版本變動，請下載 Windows 安裝版後放到腳本同資料夾，必要時改名成 `LM-Studio-Setup.exe`。
- VoiceMeeter 官網目前提供的是 ZIP 套件，不一定直接叫 `VoiceMeeterSetup.zip`。如果你下載到的是 `VoicemeeterSetup_v2122.zip`，可以直接使用，不用改名。
- SyncTranslate 如果是從 GitHub Release 下載，通常檔名會是 `SyncTranslate-onedir-windows.zip`，腳本可直接辨識。

---

## 二、你會拿到的檔案

本手冊搭配以下兩個檔案一起使用：

### 1. `install_sync_translate.ps1`
這是主要安裝腳本。  
你要用 PowerShell 執行它。

### 2. `run_install_sync_translate.cmd`
這是方便一般使用者雙擊執行的啟動檔。  
它的作用是幫你呼叫 PowerShell 去執行 `.ps1`。

---

## 三、建議的安裝資料夾

本手冊預設會把資料放在：

```text
C:\Tools\SyncTranslate\
```

腳本會建立這些資料夾：

```text
C:\Tools\SyncTranslate\
├─ Downloads
├─ Temp
└─ SyncTranslate
```

說明：
- `Downloads`：放安裝檔、壓縮檔
- `Temp`：暫存用
- `SyncTranslate`：放解壓後的程式本體

---

## 四、正式安裝步驟

---

### 步驟 1：下載本手冊附的兩個檔案

請把以下兩個檔案放在同一個資料夾，例如桌面上的 `SyncTranslateSetup` 資料夾：

- `install_sync_translate.ps1`
- `run_install_sync_translate.cmd`

---

### 步驟 2：把安裝檔放到同一個資料夾

請確認以下五個檔案都在同一個資料夾中，例如桌面上的 `SyncTranslateSetup`：

- `install_sync_translate.ps1`
- `run_install_sync_translate.cmd`
- `LM-Studio-Setup.exe`
- `VoiceMeeterSetup.zip` 或 `VoicemeeterSetup_v2122.zip`
- `SyncTranslate-onedir.zip` 或 `SyncTranslate-onedir-windows.zip`

如果你的 SyncTranslate 不是 zip，而是一整包資料夾，建議先壓成 `SyncTranslate-onedir.zip` 再交給腳本處理。

---

### 步驟 3：以系統管理員身分執行安裝

有兩種方式。

#### 方式 A：直接雙擊執行
1. 右鍵點 `run_install_sync_translate.cmd`
2. 選擇 **以系統管理員身分執行**

#### 方式 B：用 PowerShell 執行
1. 開啟 PowerShell（建議用系統管理員）
2. 切換到檔案所在資料夾
3. 執行：

```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force
.\install_sync_translate.ps1
```

---

### 步驟 4：等待腳本完成自動處理

腳本執行時，會依序做這些事情：

1. 建立安裝資料夾
2. 整理 LM Studio、VoiceMeeter、SyncTranslate 安裝檔
3. 啟動 LM Studio 安裝程式
4. 啟動 VoiceMeeter 安裝程式
5. 解壓 SyncTranslate onedir 到指定資料夾
6. 嘗試開啟 LM Studio
7. 顯示後續要你手動完成的提醒

---

## 五、安裝腳本跑完後，你接下來要做什麼

這一段很重要。  
**腳本執行完，不代表整個系統已經可以用了。**  
你還需要照下面步驟完成設定。

---

### 步驟 5：完成 LM Studio 安裝

如果腳本已經幫你啟動 LM Studio 安裝程式：

1. 依畫面指示完成安裝
2. 安裝完成後啟動 LM Studio

如果安裝過程有要求你同意授權、選擇路徑或按 Next，照畫面操作即可。

---

### 步驟 6：在 LM Studio 下載模型

打開 LM Studio 後，照以下步驟做：

1. 進入 `Discover`
2. 搜尋：`hy-mt1.5-7b`
3. 找到 **Tencent** 作者發佈的版本
4. 選擇量化版本：`Q4_K_M`
5. 按下載
6. 等下載完成

> 如果搜尋結果很多，請仔細確認：
> - 模型名稱：`hy-mt1.5-7b`
> - 作者：`Tencent`
> - 量化：`Q4_K_M`

---

### 步驟 7：在 LM Studio 載入模型

模型下載完成後：

1. 切到 `Chat` 或 `Developer`
2. 選擇剛下載好的 `hy-mt1.5-7b`
3. 將模型載入到本機記憶體中

如果沒有載入模型，SyncTranslate 之後通常會抓不到可用模型。

---

### 步驟 8：啟用 LM Studio 的 Local Server

在 LM Studio 中：

1. 進入 `Developer`
2. 找到 Local Server 或 OpenAI 相容 API 相關設定
3. 啟用本機伺服器
4. 確認位址為：

```text
http://127.0.0.1:1234
```

這一步非常重要。  
因為 SyncTranslate 會透過這個位址去呼叫 LM Studio。

---

### 步驟 9：完成 VoiceMeeter 安裝

如果安裝過程中有跳出驅動或音訊裝置安裝提示，請接受並完成。  
有些情況下，安裝完 VoiceMeeter 可能需要 **重新開機**。

如果系統要求重新開機：
1. 先存好目前工作
2. 重新開機
3. 開機後再回來繼續下面步驟

---

### 步驟 10：確認 SyncTranslate 已經被解壓完成

預設情況下，SyncTranslate 會被解壓到：

```text
C:\Tools\SyncTranslate\SyncTranslate
```

請進去這個資料夾確認是否看得到：
- `SyncTranslate.exe`
- 其他 onedir 必要檔案與子資料夾

如果沒有看到 `SyncTranslate.exe`，表示：
- 下載的壓縮檔可能不是正確版本
- 壓縮檔結構不符合預期
- 解壓失敗

---

### 步驟 11：啟動 SyncTranslate

在 `SyncTranslate.exe` 上雙擊執行。

如果你希望以後更方便，建議你：
- 對 `SyncTranslate.exe` 按右鍵
- 選擇「傳送到桌面建立捷徑」

---

## 六、第一次打開 SyncTranslate 後要做的設定

---

### 步驟 12：設定 LLM 模型

打開 SyncTranslate 後：

1. 進入 `設定`
2. 展開 `顯示進階設定`
3. 找到 `翻譯模型 LLM`
4. 確認後端是 `LM Studio`
5. 按 `重新載入模型`
6. 在本地方向與遠端方向分別選擇：
   - `hy-mt1.5-7b`

如果你只載入一個模型，兩個方向先選同一個即可。

---

### 步驟 13：設定音訊裝置

進入：

```text
設定 > 音訊裝置
```

然後設定這四項：

- `遠端輸入`
- `遠端輸出`
- `本地輸入`
- `本地輸出`

這一段通常要依你的實際設備、耳機、麥克風與 VoiceMeeter 配置而定。  
如果你完全沒有概念，建議你先記住一個原則：

- **本地輸入**：通常是你的麥克風
- **本地輸出**：通常是你自己聽得到的耳機或喇叭
- **遠端輸入 / 輸出**：通常會搭配 VoiceMeeter 虛擬裝置做轉接

---

### 步驟 14：先做系統檢查

在 `設定` 頁按下：

```text
系統檢查
```

確認以下三項都正常：
- ASR 正常
- LLM 正常
- TTS 正常

如果這三項有任何一項異常，先不要急著正式使用，先排除問題。

---

## 七、第一次功能測試

---

### 步驟 15：回到即時字幕頁做基本測試

1. 回到 `即時字幕`
2. 先把：
   - 遠端輸出模式切成 `直通`
   - 本地輸出模式切成 `翻譯`
3. 說一小段話
4. 確認：
   - 有沒有出現字幕
   - 有沒有出現翻譯字幕
   - 有沒有聲音輸出
   - 輸出音量是否正常

---

## 八、常見問題排除

### 問題 1：LM Studio 裝好了，但 SyncTranslate 抓不到模型
請依序檢查：
1. 模型是否真的已下載完成
2. 模型是否真的已載入
3. Local Server 是否已開啟
4. API 位址是否是 `http://127.0.0.1:1234`
5. SyncTranslate 中是否有按 `重新載入模型`

---

### 問題 2：第一次跑很慢
這通常是正常的，原因可能包括：
- LM Studio 剛載入模型
- Whisper 權重第一次下載
- 首次建立快取
- VoiceMeeter 剛初始化

第一次通常會比較慢，第二次之後會改善。

---

### 問題 3：只有字幕，沒有聲音
請檢查：
1. 模式是否設對（直通 / 翻譯）
2. 輸出音量是否太低
3. 音訊裝置是否選錯
4. VoiceMeeter 的路由是否正確
5. TTS 聲線是否可用

---

### 問題 4：VoiceMeeter 安裝完後裝置沒出現
請先試：
1. 重新開機
2. 再開啟 VoiceMeeter
3. 再到 Windows 聲音設定中查看裝置是否已出現

---

## 九、給一般使用者的最短版流程

如果你之後要教其他人，可以直接講這段：

1. 把安裝腳本、啟動檔、LM Studio 安裝檔、VoiceMeeter 安裝檔、`SyncTranslate-onedir.zip` 放同一個資料夾  
2. 以系統管理員執行 `run_install_sync_translate.cmd`  
3. 等腳本幫你安裝 LM Studio、VoiceMeeter、解壓 SyncTranslate  
4. 手動打開 LM Studio，下載 `hy-mt1.5-7b`（Tencent，`Q4_K_M`）  
5. 在 LM Studio 開啟 Local Server（`http://127.0.0.1:1234`）  
6. 打開 SyncTranslate，設定模型與音訊裝置  
7. 跑系統檢查  
8. 完成

---

## 十、附註：這份手冊的定位

這份手冊是為了讓安裝流程從「很多零碎步驟」變成「盡量集中由腳本處理」。  
但你仍應該把它視為：

- **自動安裝 + 人工設定引導手冊**

而不是完全無人值守安裝。

如果未來你要做到更進一步的全自動，通常還需要：
- 可穩定下載的官方安裝檔 URL
- 可合法自動化的模型下載來源
- 更完整的靜默安裝參數
- VoiceMeeter 與 Windows 音訊裝置的預設路由自動設定方案

在目前情況下，這份手冊已經是很適合交給一般使用者使用的版本。
