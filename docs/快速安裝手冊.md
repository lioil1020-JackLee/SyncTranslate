# 快速安裝手冊

這份文件提供 Windows 本機安裝 SyncTranslate 的最短流程。

## 建議環境

- Windows 10 / 11
- Python 與 `uv`
- VoiceMeeter（音訊路由）
- 建議有可用 CUDA 的 NVIDIA GPU（CPU 也可運作，但延遲較高）

## 最短安裝步驟

> **一鍵安裝（推薦）**：若是全新環境，可直接執行以下腳本完成所有步驟（2–4）：
> ```powershell
> .\tools\runtime_setup\setup_complete.ps1
> ```
> 腳本會自動：檢查 Visual Studio Build Tools → 準備外部運行時 → 下載 LLM 模型 → 安裝 llama-cpp-python → 驗證所有依賴。

### 1. 安裝相依套件

```powershell
uv sync --locked
```

### 2. 準備外部 AI 運行時

ASR（faster-whisper）、共用計算套件與翻譯用 llama.cpp in-process engine 已外部化管理：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\runtime_setup\prepare_external_runtimes.ps1
```

> **如果 Build Tools 未安裝**：此步驟需要編譯 llama-cpp-python（C++ 擴展）。
> 1. 前往 https://visualstudio.microsoft.com/visual-cpp-build-tools/ 下載安裝程式
> 2. 選擇 **Desktop development with C++** 工作負載（約 2GB）
> 3. 安裝完成後**重啟電腦**（使 `nmake` 在 PATH 中生效）
> 4. 重新執行上述腳本

此步驟會在 `runtimes/` 下建立隔離的運行時與模型目錄：
- `runtimes/shared/` - torch, torchaudio, onnxruntime, llama-cpp-python（共用）
- `runtimes/faster_whisper/` - faster-whisper, ctranslate2, tiktoken（ASR）
- `runtimes/models/llm/hy-mt1.5-7b.gguf` - 翻譯模型

中文 ASR 模型預設路徑為 `runtimes/models/belle-zh-ct2`（需另行下載或轉換）。  
非中文 ASR 使用 faster-whisper `large-v3-turbo`（首次使用時自動下載）。
翻譯 GGUF 會自動從 `tencent/HY-MT1.5-7B-GGUF` 下載 `HY-MT1.5-7B-Q4_K_M.gguf`。若要使用自備 GGUF，可改用 `-LlmModelPath "D:\models\hy-mt1.5-7b.gguf"`。

### 3. 確認本機翻譯 runtime

SyncTranslate 使用 `llama-cpp-python` 將 GGUF in-process 載入主程式，無需任何外部 server。確認 `runtimes/models/llm/hy-mt1.5-7b.gguf` 存在即可。

### 4. 準備 VoiceMeeter

- 建立你要的虛擬輸入 / 輸出 routing
- 確認 SyncTranslate 要使用的輸入與輸出裝置都能在 Windows 裡看到

### 5. 啟動 SyncTranslate

```powershell
uv run python .\main.py
```

### 6. 先跑一次健康檢查

```powershell
uv run python .\main.py --check
```

## 進程內設定順序

1. 到 `字幕頁` 設定每通道 ASR 語言
2. 到 `Local AI` 頁選擇使用情境：
   - `會議模式（穩定切段）`：適合會議、影片、長句內容，VAD 較保守
   - `對話模式（低延遲）`：適合短句雙向對話，切段更快
3. ASR 模型已自動路由，**不需手動選擇**：
   - 中文通道固定使用 `belle-zh-ct2`
   - 非中文通道固定使用 `large-v3-turbo`
4. 確認裝置、LLM runtime / TTS 設定
5. 到 `音訊路由` 頁確認輸入與輸出裝置
6. 開始測試

## ASR 路由重點

ASR 模型由系統根據通道語言自動決定，不需手動選 engine：

- 中文（`zh-TW` / `zh-CN` 等）-> `belle-zh-ct2` + `silero-vad`
- 非中文 / `auto` -> `large-v3-turbo` + `silero-vad`
- `none` -> 停用該通道 ASR

## 第一輪驗收建議

建議先測：

1. 單通道中文
2. 單通道非中文
3. 雙通道同時發話

測完後看 `logs/session_reports/*.json`：

- `resolved_backend`
- `device_effective`
- `endpoint_signal.pause_ms`
- `speech_started_count`
- `soft_endpoint_count`
- `hard_endpoint_count`
- `partial_count`
- `final_count`
- `postprocessor.final.rejected_count`
- `postprocessor.final.last_rejection_reason`
- `dropped_chunks`

## 驗證安裝

```powershell
# 驗證翻譯 runtime
.\runtimes\shared\Scripts\python.exe -c "from app.infra.translation.provider import create_translation_provider; from app.infra.config.schema import LlmConfig; p = create_translation_provider(LlmConfig()); ok, msg = p.health_check(); print('Translation ready!' if ok else f'Not ready: {msg}')"

# 驗證 CUDA Torch
.\runtimes\shared\Scripts\python.exe -c "import torch; print(f'torch={torch.__version__}, cuda={torch.cuda.is_available()}')"
```

## 常見問題

### Build Tools 相關錯誤

| 錯誤訊息 | 解決方式 |
|----------|----------|
| `CMake Error: CMAKE_C_COMPILER not set` | 安裝 Visual Studio Build Tools（C++ 工作負載） |
| `no such file or directory (nmake)` | 同上，並重啟電腦 |
| `llama-cpp-python installation failed` | 確認 Build Tools 完整安裝後重新執行腳本 |

```powershell
# 驗證 nmake 是否可用
where nmake
```

若有返回路徑 → Build Tools 已正確安裝；若無返回 → 重新安裝並選擇 C++ 工作負載。

### 啟動很慢

現在模型已改成懶載入。冷啟動通常比以前快，但第一次真正啟用中文 ASR 時，仍會載入 faster-whisper / silero-vad。

### 明明選了 CUDA，卻還是很慢

因為 `device: cuda` 只是 requested device。  
真正有沒有跑到 GPU，要看 diagnostics / session report 裡的：

- `device_effective`

如果它是 `cpu`，代表目前環境沒有真的用到 CUDA。

### 本地與遠端同時中文發話時延遲升高

這通常表示雙通道都走 `faster-whisper`，但實際 effective device 仍是 `CPU`。  
這時不是 UI 設錯，而是執行資源不足。

### 驗證 CUDA Torch

外部運行時使用 CUDA 版本（`torch==2.10.0+cu128`）。驗證方式：

```powershell
# 檢查 shared 運行時中的 torch
.\runtimes\shared\Scripts\python.exe -c "import torch; print(f'torch={torch.__version__}, cuda={torch.cuda.is_available()}')"
```

若 `cuda=False`，代表該環境無 NVIDIA GPU 或 CUDA 驅動未正確安裝。

## 開發工具

### ASR Benchmark

離線 ASR 品質量測工具，評估 CER / WER 與延遲：

```powershell
# 執行 benchmark（結果輸出為 jsonl）
uv run python tools/asr_benchmark/run_benchmark.py \
  --audio <音訊檔路徑> --profile default --reference <參考文字> \
  --chunk-ms 40 --output downloads/benchmark_results/result.jsonl

# 產生可讀報表
uv run python tools/asr_benchmark/report.py \
  --input downloads/benchmark_results/result.jsonl --format table
```

### YouTube SRT 基準測試

用 YouTube 字幕作為 ground truth 評估 ASR 品質：

```powershell
# 下載中文字幕集
uv run python tools/youtube_srt/download_chinese.py

# 下載英文字幕集
uv run python tools/youtube_srt/download_english.py

# 對指定影片進行 benchmark
uv run python tools/youtube_srt/benchmark_against_yt.py \
  --url "https://www.youtube.com/watch?v=<VIDEO_ID>" \
  --source remote --lang en --profile default \
  --out-dir downloads/benchmark \
  --output downloads/benchmark_results/result.json
```

字幕快取位置：`downloads/chinese_srt/`、`downloads/english_srt/`。  
Benchmark 結果位置：`downloads/benchmark_results/`。

## 打包 Onedir（發佈版本）

完整打包流程（假設已執行過 `prepare_external_runtimes.ps1`）：

```powershell
# 1. 確保 build 工具已安裝
uv sync --locked --group build

# 2. 構建 onedir（會自動排除 torch/faster-whisper/faster-whisper 等，改由外部運行時提供）
uv run pyinstaller .\SyncTranslate-onedir.spec --noconfirm --clean

# 3. 複製開發環境的外部運行時到打包結果
powershell -ExecutionPolicy Bypass -File .\tools\runtime_setup\relocate_ai_runtime_artifacts.ps1

# 4. 壓縮 onedir（需要 7-Zip；腳本會找 PATH 或 C:\Program Files\7-Zip\7z.exe）
powershell -ExecutionPolicy Bypass -File .\tools\runtime_setup\package_onedir.ps1 -Source .\dist\SyncTranslate-onedir -Output .\dist\SyncTranslate-onedir-windows.zip
```

最終輸出位置：`dist/SyncTranslate-onedir/`

relocate 腳本會強制檢查 `runtimes/shared` 的 `llama_cpp` 套件與 `runtimes/models/llm/hy-mt1.5-7b.gguf`，缺檔會直接中止，避免打出缺翻譯 runtime 的包。
它只複製必要 runtime 與模型，不會把本機 Hugging Face cache 一起打包；非中文 ASR 若未隨包提供 cache，首次使用時仍會依 faster-whisper 機制下載。

目錄結構：
```
dist/SyncTranslate-onedir/
├── SyncTranslate.exe           # 主程式
├── config.yaml                 # 設定檔
├── runtimes/                   # 外部 AI 運行時（從 dev 複製）
│   ├── shared/Lib/site-packages/   # torch, onnxruntime, llama-cpp-python
│   ├── faster_whisper/Lib/site-packages/
│   └── models/
│       ├── belle-zh-ct2/
│       └── llm/hy-mt1.5-7b.gguf
└── _internal/                  # PyInstaller 主應用依賴
```
