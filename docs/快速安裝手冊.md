# SyncTranslate 快速安裝手冊（全自動部署版）

這份手冊的目標是讓第一次接觸電腦音訊、LM Studio、CUDA、VoiceMeeter 的使用者，也能把 `SyncTranslate` 安裝完成。

你可以把它理解成：

- 自動化腳本會先幫你處理「資料夾建立、安裝檔整理、啟動安裝器、解壓 SyncTranslate」。
- 少數無法保證完全自動的部分，例如顯示卡驅動、Windows 權限確認、VoiceMeeter 安裝後可能要求重開機、LM Studio 第一次登入與模型下載，手冊會一步一步告訴你怎麼做。

如果你完全不懂這些工具，也請不要跳步，照順序做就好。

---

## 你最後會得到什麼

完成後，你的電腦會有：

- `LM Studio`
- `VoiceMeeter`
- `SyncTranslate`
- 一個可供 SyncTranslate 使用的本地翻譯模型
- SyncTranslate 可連線的本地 API 位址：`http://127.0.0.1:1234`

---

## 先講結論：哪些步驟是自動的，哪些不是

### 腳本會自動做的事

- 建立安裝資料夾
- 找出你放在同資料夾內的安裝檔
- 啟動 `LM Studio` 安裝程式
- 啟動 `VoiceMeeter` 安裝程式
- 解壓 `SyncTranslate-onedir.zip`
- 嘗試開啟 `LM Studio`
- 告訴你 `SyncTranslate.exe` 在哪裡

### 腳本可能無法完全自動完成的事

- NVIDIA 顯示卡驅動安裝或更新
- CUDA / GPU 驅動環境異常排除
- Windows 的系統管理員權限確認視窗
- `LM Studio` 第一次登入或第一次啟動初始化
- 在 `LM Studio` 下載模型
- 在 `LM Studio` 載入模型
- 在 `LM Studio` 啟用 Local Server
- `VoiceMeeter` 安裝後重新開機
- 最後的音訊裝置路由選擇

這不是你操作錯誤，而是這些步驟本來就會受到每台電腦的硬體、權限、Windows 狀態影響。

---

## 安裝前必讀

### 1. 適用環境

建議至少符合以下條件：

- Windows 10 或 Windows 11
- NVIDIA 顯示卡
- 至少 16 GB RAM
- 至少 6 GB VRAM
- 穩定網路
- 可使用系統管理員權限

如果你沒有 NVIDIA 顯示卡，或顯卡驅動太舊，成功率會明顯下降。

### 2. 本手冊預設使用的模型

本手冊以 `hy-mt1.5-7b` 的 `Tencent` 版本、量化為 `Q4_K_M` 為例。

如果之後你改用別的模型也可以，但第一次部署請先照這份手冊，不要自行更換，先把整套流程跑通。

### 3. 你需要準備的檔案

把以下檔案放在同一個資料夾，例如桌面的 `SyncTranslateSetup`：

- `install_sync_translate.ps1`
- `run_install_sync_translate.cmd`
- `LM-Studio-Setup.exe`
- `VoiceMeeterSetup.zip` 或 `VoicemeeterSetup_v2122.zip`
- `SyncTranslate-onedir.zip` 或 `SyncTranslate-onedir-windows.zip`

建議資料夾看起來像這樣：

```text
SyncTranslateSetup
├─ install_sync_translate.ps1
├─ run_install_sync_translate.cmd
├─ LM-Studio-Setup.exe
├─ VoiceMeeterSetup.zip
└─ SyncTranslate-onedir-windows.zip
```

### 4. 如果你還沒下載安裝檔

請先分別下載：

- LM Studio Windows 安裝版
- VoiceMeeter 安裝包
- SyncTranslate onedir 發行包

下載後放回同一個 `SyncTranslateSetup` 資料夾即可。

如果檔名不同：

- LM Studio 請改成 `LM-Studio-Setup.exe`
- SyncTranslate zip 檔可保留 `SyncTranslate-onedir-windows.zip`
- VoiceMeeter zip 檔可保留原始官方檔名

## 第 0 步：先確認 NVIDIA 驅動與 CUDA 狀態

這一步很重要。原本手冊少了這段，但對新手來說，這其實是最容易卡住的地方。

### 0-1. 怎麼判斷你有沒有 NVIDIA 顯示卡

在 Windows 搜尋列輸入：

```text
裝置管理員
```

打開後找到：

```text
顯示卡
```

如果你看到類似：

- `NVIDIA GeForce RTX ...`
- `NVIDIA RTX ...`

代表可以繼續。

如果你完全沒有看到 NVIDIA，只看到 Intel 或 AMD：

- 這份流程不建議繼續
- 請先改成 CPU 測試路線，或換有 NVIDIA 顯卡的電腦

### 0-2. 怎麼快速檢查驅動是否正常

按 `Win + R`，輸入：

```text
cmd
```

在黑色視窗輸入：

```text
nvidia-smi
```

#### 如果有正常顯示

你會看到 NVIDIA 驅動版本、GPU 名稱、記憶體等資訊。

這代表：

- 顯示卡驅動大致正常
- 可以繼續執行安裝

#### 如果出現「不是內部或外部命令」或錯誤

表示很可能：

- 沒安裝 NVIDIA 驅動
- 驅動安裝失敗
- 驅動壞掉
- 這台電腦根本不是 NVIDIA 顯卡

這時請先做下面的驅動安裝流程。

---

## 第 1 步：安裝或更新 NVIDIA 顯示卡驅動

### 1-1. 建議做法

到 NVIDIA 官方網站下載對應你顯卡的最新版驅動，安裝完成後再回來。

如果你不知道該選哪個版本，保守做法是：

- GeForce 系列選 Game Ready Driver
- RTX 專業卡或工作站卡選對應的官方驅動

### 1-2. 安裝時怎麼選

驅動畫面如果出現安裝選項：

- 選 `Express` 或 `快速安裝`：適合大部分新手
- 若你曾經有很多顯卡問題，才考慮 `Custom` 或 `自訂`

### 1-3. 安裝過程中可能會發生什麼

- 螢幕閃爍幾次
- 視窗短暫黑畫面
- 安裝需要幾分鐘
- 安裝後要求重新開機

這些都很正常。

### 1-4. 如果它要求重新開機

請直接重開。

重開後第一件事：

1. 再次打開 `cmd`
2. 再輸入 `nvidia-smi`
3. 確認有正常顯示

確認正常後，再繼續下面的安裝步驟。

---

## 第 2 步：執行全自動部署腳本

### 2-1. 執行方式

在 `SyncTranslateSetup` 資料夾中：

1. 右鍵 `run_install_sync_translate.cmd`
2. 選 `以系統管理員身分執行`

如果看到 Windows 的權限提示：

- 選 `是`

### 2-2. 腳本執行後會做什麼

它會依序：

1. 建立 `C:\Tools\SyncTranslate`
2. 複製或整理安裝檔
3. 啟動 LM Studio 安裝程式
4. 啟動 VoiceMeeter 安裝程式
5. 解壓 SyncTranslate
6. 嘗試打開 LM Studio

### 2-3. 執行後你可能會看到很多視窗

這是正常的，請不要把它們全部關掉。

你要做的是：

- 跟著安裝器按 `Next`
- 遇到授權條款按 `I Agree`、`Accept` 或 `同意`
- 沒把握時，保持預設選項即可

---

## 第 3 步：完成 LM Studio 安裝

### 3-1. 如果跳出 LM Studio 安裝視窗

請照這樣做：

1. 一直按 `Next`
2. 安裝位置沒特殊需求就不要改
3. 按 `Install`
4. 安裝完成後按 `Finish`

### 3-2. 如果安裝完成後沒有自動打開 LM Studio

請手動從 Windows 開始選單打開：

```text
LM Studio
```

### 3-3. LM Studio 第一次開啟時

有時會遇到以下情況：

- 首次初始化
- 要求登入或跳過
- 要求允許網路
- 要求選擇模型儲存位置

建議做法：

- 可以登入，也可以先跳過
- 如果 Windows 防火牆跳提示，允許存取
- 模型儲存位置若你不確定，就用預設

只要最後能正常進入 LM Studio 主畫面就可以。

---

## 第 4 步：在 LM Studio 下載模型

### 4-1. 進入模型搜尋

在 LM Studio 裡：

1. 找到 `Discover`
2. 在搜尋框輸入：

```text
hy-mt1.5-7b
```

### 4-2. 選哪一個

優先選：

- 模型名稱包含 `hy-mt1.5-7b`
- 發布者或來源是 `Tencent`
- 量化格式選 `Q4_K_M`

### 4-3. 如果同名模型很多個

請照這個順序判斷：

1. 先選 `Tencent`
2. 再選 `Q4_K_M`
3. 如果還是很多，先選下載量高或比較常見的版本

### 4-4. 下載中要注意什麼

模型檔通常很大，請注意：

- 不要關閉 LM Studio
- 不要讓電腦睡眠
- 網路中斷會導致下載失敗

### 4-5. 如果下載失敗

請這樣做：

1. 確認網路正常
2. 關閉 LM Studio 再重開
3. 回到 `Discover` 重新下載

若磁碟空間不足，先清出空間再重新下載。

---

## 第 5 步：在 LM Studio 載入模型

模型下載完成後，請把它載入。

通常作法是：

1. 切到 `Chat` 或模型頁面
2. 選取剛剛下載好的 `hy-mt1.5-7b`
3. 按 `Load`

### 成功的判斷方式

如果模型成功載入，通常會看到：

- 有載入中的進度
- 載入完成後可開始輸入文字
- 不再顯示尚未載入模型

### 如果載入失敗

最常見原因：

- VRAM 不夠
- 驅動有問題
- 模型檔下載不完整

建議排除順序：

1. 先重開 LM Studio
2. 再試一次 `Load`
3. 如果還是不行，改用更小的模型或更低量化版本

---

## 第 6 步：在 LM Studio 啟用 Local Server

這一步很重要，因為 `SyncTranslate` 是透過 API 連到 LM Studio。

### 6-1. 打開 Local Server

在 LM Studio 中：

1. 找到 `Developer`
2. 找到 `Local Server`
3. 啟動 server

### 6-2. 確認位址

請確認位址是：

```text
http://127.0.0.1:1234
```

如果不是這個位址，請改成這個。

### 6-3. 防火牆視窗怎麼選

如果 Windows 防火牆跳出提示：

- 選 `允許存取`

如果有網路類型選擇：

- 一般建議至少勾 `私人網路`
- 不確定時保守選 `私人網路`

### 6-4. 這一步完成的判斷方式

只要你看到：

- Local Server 已啟動
- 位址是 `http://127.0.0.1:1234`

就可以進到下一步。

---

## 第 7 步：完成 VoiceMeeter 安裝

### 7-1. 如果跳出 VoiceMeeter 安裝程式

請照著做：

1. 按 `Install`
2. 遇到權限提示按 `是`
3. 等安裝完成

### 7-2. 如果安裝後要求重新開機

這裡請不要跳過。

請直接：

1. 儲存你手上其他工作
2. 重新開機

### 7-3. 重新開機後要做什麼

重開後請照這個順序：

1. 打開 `LM Studio`
2. 確認模型還能 `Load`
3. 確認 Local Server 仍在 `http://127.0.0.1:1234`
4. 到 `C:\Tools\SyncTranslate\SyncTranslate` 找 `SyncTranslate.exe`

如果你找不到 `SyncTranslate.exe`，請回到：

```text
C:\Tools\SyncTranslate
```

確認是不是解壓到了子資料夾內。

---

## 第 8 步：啟動 SyncTranslate

打開：

```text
C:\Tools\SyncTranslate\SyncTranslate
```

然後雙擊：

```text
SyncTranslate.exe
```

如果 Windows 跳出警告：

- 選 `更多資訊`
- 再選 `仍要執行`

如果防火牆跳提示：

- 選 `允許存取`

---

## 第 9 步：在 SyncTranslate 內設定 LLM

打開 SyncTranslate 後：

1. 進入 `設定`
2. 找到 `本地 AI` 或 LLM 設定區
3. 確認後端是 `LM Studio`
4. 確認 API 位址是：

```text
http://127.0.0.1:1234
```

5. 按 `重新載入模型`
6. 從清單選剛剛在 LM Studio 載入的模型

### 如果重新載入模型後沒有出現

請照這個順序檢查：

1. LM Studio 的模型是否真的已經 `Load`
2. Local Server 是否真的已開啟
3. 位址是否真的是 `http://127.0.0.1:1234`
4. LM Studio 是否被你關掉了

---

## 第 10 步：設定音訊裝置

這一步是整個流程裡最容易因為每台電腦不同而有差異的部分，所以請慢慢來。

### 10-1. 先記住四個方向

你通常會看到四個裝置欄位：

- 遠端輸入
- 本地輸入
- 本地輸出
- 遠端輸出

### 10-2. 新手保守設定法

如果你暫時只想先確認程式有沒有跑起來，建議先這樣：

- 本地輸入：你的實體麥克風
- 本地輸出：你的耳機或喇叭
- 遠端輸入：VoiceMeeter 的虛擬輸入
- 遠端輸出：VoiceMeeter 的虛擬輸出

### 10-3. 如果你不知道哪個是 VoiceMeeter

通常名稱會包含：

- `VoiceMeeter Input`
- `VoiceMeeter Output`
- `VoiceMeeter AUX Input`
- `VoiceMeeter AUX Output`

### 10-4. 如果安裝後完全看不到 VoiceMeeter 裝置

請依序做：

1. 重新開機
2. 打開 VoiceMeeter
3. 打開 Windows 音效設定重新確認
4. 再回 SyncTranslate 檢查

---

## 第 11 步：做系統檢查

在 SyncTranslate 內：

1. 找到 `系統檢查`
2. 執行檢查

你希望看到：

- ASR 正常
- LLM 正常
- TTS 正常

### 如果不是全部正常

請用下面方式判斷：

- `ASR 失敗`：通常是麥克風、GPU、模型或權限問題
- `LLM 失敗`：通常是 LM Studio 模型未載入或 Local Server 未開
- `TTS 失敗`：通常是語音輸出引擎或音訊裝置設定問題

先把 `LLM` 修好，再回頭調其他項目。

---

## 第 12 步：做第一次實測

建議第一次測試用最簡單的方法：

1. 戴上耳機
2. 用麥克風說一句簡單中文
3. 看本地字幕有沒有出現
4. 看翻譯後字幕有沒有出現

如果你要測雙向流程，再讓會議端音訊進來測遠端通道。

## 最常見失敗情況與處理方式

### 問題 1：腳本執行後沒有任何反應

請檢查：

1. 你是不是用 `系統管理員` 執行
2. 安裝檔是不是都放在同一個資料夾
3. 檔名是不是差太多，導致腳本找不到

最保險的作法是把檔名改成手冊前面列出的名稱。

### 問題 2：`nvidia-smi` 不能用

代表顯卡驅動沒有就緒。

請先回到「第 1 步：安裝或更新 NVIDIA 顯示卡驅動」。

### 問題 3：LM Studio 裝好了，但 SyncTranslate 抓不到模型

請依序檢查：

1. 模型是否已下載完成
2. 模型是否已在 LM Studio 內 `Load`
3. Local Server 是否已開啟
4. 位址是否是 `http://127.0.0.1:1234`
5. 在 SyncTranslate 中是否按過 `重新載入模型`

### 問題 4：LM Studio 能開，但模型載不動

最常見原因是 VRAM 不夠。

建議：

1. 關掉其他吃 GPU 的程式
2. 重開 LM Studio
3. 換更小模型
4. 換更低量化版本

### 問題 5：VoiceMeeter 安裝完沒有裝置

請照這個順序：

1. 重新開機
2. 打開 VoiceMeeter
3. 再打開 Windows 音效設定
4. 再打開 SyncTranslate 檢查

### 問題 6：SyncTranslate 打不開

請檢查：

1. `SyncTranslate.exe` 是否真的存在
2. Windows 是否攔截執行
3. 防毒軟體是否隔離檔案

### 問題 7：系統檢查 LLM 失敗

請先做這四件事：

1. 打開 LM Studio
2. 載入模型
3. 啟動 Local Server
4. 回 SyncTranslate 重按 `重新載入模型`

---

## 建議的新手安裝順序

如果你不想看太多細節，照這個順序做就可以：

1. 確認有 NVIDIA 顯卡
2. 在 `cmd` 輸入 `nvidia-smi`
3. 如果不正常，先安裝或更新驅動並重開機
4. 把所有安裝檔放到 `SyncTranslateSetup`
5. 以系統管理員執行 `run_install_sync_translate.cmd`
6. 完成 LM Studio 安裝
7. 完成 VoiceMeeter 安裝
8. 若要求重開機就立刻重開
9. 打開 LM Studio，下載 `hy-mt1.5-7b` 的 `Tencent` `Q4_K_M`
10. 載入模型
11. 啟用 Local Server `http://127.0.0.1:1234`
12. 打開 `SyncTranslate.exe`
13. 在 SyncTranslate 重新載入模型
14. 設定音訊裝置
15. 跑系統檢查
16. 做第一次實測

## 這份手冊的保證範圍

這份手冊已經把目前流程中：

- 能自動化的部分
- 不能保證自動化的部分
- 新手最常卡住的回應方式
- 需要重新開機的情況

都整理進來了。

如果你完全照著做，仍然失敗，通常不是你操作錯，而是以下其中一項環境因素：

- 顯卡驅動異常
- VRAM 不足
- Windows 音訊裝置狀態異常
- 防火牆或防毒攔截
- 模型下載不完整

遇到這類問題時，請優先回頭檢查：

1. `nvidia-smi`
2. LM Studio 模型是否已載入
3. Local Server 是否在 `http://127.0.0.1:1234`
4. VoiceMeeter 是否已安裝並在 Windows 中出現

只要這四項正常，SyncTranslate 大多就能順利跑起來。
