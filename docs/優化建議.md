## 一、仍存在的結構問題

### 1.1 `stream_worker.py`（510 行）— 技術負債最重的一個檔案

**現況：**  
`stream_worker.py` 在架構說明中已定位為「最小 compatibility helper」，但實際上仍包含大量主動代碼：

- `StreamingAsr` class（~250 行）— 仍含完整的 adaptive tuning 邏輯，測試（`test_multilingual_channel_policy.py`）仍直接使用它
- 4 個模組級幻覺過濾函數（`_looks_like_silence_hallucination`、`_transcript_drop_reason`、`_looks_like_known_non_speech_text`、`_looks_like_script_mismatch_junk`）
- 5 個 Unicode 輔助函數（`_is_latin`、`_is_cjk`、`_is_hangul`、`_is_japanese`、`_contains_cyrillic_or_greek`）

**問題：**
1. 幻覺過濾邏輯**已分裂在兩個地方**：`_hallucination_filter.py`（tail hallucination）和 `stream_worker.py`（silence hallucination、non-speech overlay、script mismatch）。兩者的職責邊界模糊，難以維護。
2. Unicode 輔助函數（`_is_cjk`、`_is_latin` 等）在以下多個檔案中**各自重新定義**：
   - `stream_worker.py`
   - `text_correction.py`（`_contains_cjk`、`_is_cjk_char`）
   - `_stream_parser.py`（`_contains_cjk`）
   - `lexical_bias_v2.py`（`_is_cjk_text`）
   - `transcript_validator_v2.py`（`_is_cjk_family_language`）

**建議：**
```
app/infra/asr/_hallucination_filter.py   ← 合入 stream_worker.py 的 3 個幻覺過濾函數
app/domain/unicode_utils.py              ← 新建：集中所有 CJK/Unicode 判斷 helper
```
合併後 `stream_worker.py` 應可縮減至 ~150 行的純 shim；各檔案改 import `unicode_utils`。

## 三、架構層次問題

### 3.1 `stream_worker.py` 的 `StreamingAsr` 不應由測試直接使用

`test_multilingual_channel_policy.py` 直接 import `StreamingAsr` 並且測試其 hallucination filter 函數，但 `StreamingAsr` 是已標記「舊版」的 compatibility stub。這些測試應改為直接測試 `_hallucination_filter.py` 和未來合併進去的函數，而非透過 `stream_worker`。

### 3.2 `TranslationDispatcher` 未被 `AudioRouter` 使用

`app/application/translation_dispatcher.py`（155 行）定義了 `TranslationDispatcher` class，但 `AudioRouter` 仍自帶完整的翻譯排隊 + async worker 邏輯（約 100 行），兩者責任重疊，`TranslationDispatcher` 沒有被整合進去。

**建議：** 讓 `AudioRouter` 內部使用 `TranslationDispatcher` 做翻譯 fanout，而非自行維護 translation queue/worker。

### 3.3 `asr_event_processor.py` 與 `AudioRouter._on_asr_event` 的職責重疊

`app/application/asr_event_processor.py`（160 行）定義 `AseEventProcessor`，而 `AudioRouter._on_asr_event` 直接內嵌了完整的事件處理邏輯（~90 行）。兩者都在做「收 ASR event → 決定 transcript store / TTS」的事，但沒有實際連結。

## 五、代碼品質小問題

### 5.1 `getattr` 防禦性存取過度使用

`audio_router.py` 中大量出現：
```python
runtime = getattr(self._runtime_config, "runtime", None)
value = getattr(runtime, "some_field", default) if runtime is not None else default
```
這種雙層 `getattr` 是因為 `_runtime_config` 初始為 `None`。若確保 `_runtime_config` 在 `start()` 前必定被設定，可以改為直型屬性存取，代碼更清晰。

### 5.2 `stream_worker.py` 中的 `request_stop() -> None: return None`

第 220 行的 `request_stop()` 方法只有 `return None`，是 compatibility stub 的殘留，應移除或加上 `pass`。

### 5.3 `audio_router.py` 的 `_on_asr_event` 過長（~90 行）

單一方法包含完整的 `try/except + if/else` 決策樹，建議萃取為 `_handle_asr_event_inner()` 或拆分 no-translation / translation 分支。
