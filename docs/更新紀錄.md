# 更新紀錄

## 2026-04-25（ASR 即時串流辨識率調校）

### ASR / Streaming

- 離線 ASR benchmark 已達穩定水準，後續優化重點改放即時串流。
- 新增 `app/infra/asr/resampling.py`，faster-whisper、Silero VAD 與 backend 前處理共用 16k resampling。
- 新增 `app/infra/asr/language_profiles.py`，依 ASR 語言套用不同 VAD / endpoint / prompt / no-speech 設定。
- 修正非 CJK / `auto` 通道沿用中文 initial prompt 的問題。
- `worker_v2.py` 新增 segment-local VAD / RMS 統計，final validator 可用每段音訊自己的 speech ratio。
- 新增短尾端幻聽過濾，降低空白尾音被辨識成 `You`、`Thank you`、短片尾訂閱提示的機率。
- empty-final partial fallback 會拒絕短 CTA partial，避免 final 空白時補回片尾幻聽。
- validator 會拒絕短字幕 overlay / 製作 credit（如字幕志願者、詞曲、片尾訂閱提示）。
- 中文 profile 將 `no_speech_threshold` 調整為 `0.25`，提升含背景音長故事的尾端靜音判斷。
- `asr_final_correction_enabled` 預設維持關閉；實測中文長故事中 LLM final correction 可能過度改寫。

### Benchmark / Docs

- `tools/asr_benchmark/run_benchmark.py` 與 `streaming_sim.py` 會套用語言 profile。
- `streaming_sim.py` 報表改為顯示實際 endpoint profile 與 worker timing，並分開記錄實際輸出 final count。
- 健康檢查 UI 訊息改為乾淨的繁中，整體狀態區分「正常 / 有異常 / 逾時」，ASR / LLM / TTS 子項各自顯示正常或異常。
- 健康檢查背景程序改走 `main.py --healthcheck-worker --healthcheck-config ...`，避免 `python -m` 在部分 UI 啟動環境下失敗；traceback 也會壓縮成一行摘要。
- 新增 `tools/runtime_smoke/run_runtime_smoke.py` 與 `tools/runtime_smoke/healthcheck_ui_smoke.py`，用真實 app entrypoint 補上 pytest 無法涵蓋的 UI / subprocess runtime 檢查。
- Local AI 快速模式新增 `低延遲對話（穩定 ASR）`，讓中文 ASR 可維持 `meeting_room`，同時讓 LLM / TTS 走低延遲對話設定；`低延遲雙向對話` 則明確代表 ASR `turn_taking`。
- 實測參考：
  - 英文離線 normalized WER 約 `0.8%`
  - 中文離線 normalized CER 約 `6.8%`
  - 英文即時 normalized accuracy 約 `99.3%`
  - 中文長故事即時約 `86.3%`
- 更新 `README.md`、`docs/架構說明.md`、`docs/設定說明.md`、`docs/測試說明.md`、`docs/ASR重構藍圖.md`。

## 2026-04-21（ASR v2 單一路徑、穩定化與診斷強化）

### ASR / Runtime

- `ASRManagerV2` / `SourceRuntimeV2` 成為唯一實際執行路徑；舊 `streaming_pipeline.py` 已移除。
- `stream_worker.py` 改為最小 compatibility helper，不再承載舊 worker 執行流程。
- 修正多個 runtime 設定未真正接入 v2 的問題：
  - `early_final_enabled`
  - `adaptive_asr_enabled`
  - `asr_partial_interval_floor_ms`
  - `degradation_policy_enabled`
  - endpoint finalize thresholds
- 音訊進 ASR 前固定採更穩定的 mono 策略，避免雙聲道相消或錯選聲道。

### partial / final 穩定化

- `worker_v2.py`
  - final 與最後 partial 的合併更保守
  - 新增 repetitive loop final 保護
  - 新增短 pause 的 early-final defer
- `audio_router.py`
  - stable partial 判斷改為共同前綴 + 字元差雙重檢查
- `transcript_validator_v2.py`
  - 新增 `markup-leak` 過濾，擋掉 `</solution>`、`assistant:` 等殘渣

### 預設與 UI

- UI 快速模式改名：
  - `超穩定會議字幕`
  - `低延遲雙向對話`
- `meeting_room` profile 進一步偏向長句字幕 / 會議監聽。
- `turn_taking` profile 保留低延遲，但降低過度亂切風險。
- schema / `config.example.yaml` 預設改為穩定取向：
  - `asr_profile_local = meeting_room`
  - `asr_profile_remote = meeting_room`
  - `early_final_enabled = false`
  - `stable_partial_min_repeats = 3`
  - `partial_stability_max_delta_chars = 6`
  - `asr_partial_min_audio_ms = 360`
- diagnostics 頁現在可直接看到 ASR queue、partial/final、pause、endpoint、degradation、validator reject 摘要。

### Diagnostics / Export

- `manager.stats()` / diagnostics export 補上：
  - `postprocessor.final.rejected_count`
  - `postprocessor.final.last_rejection_reason`
  - `endpoint_signal.pause_ms`
  - `speech_started_count / soft_endpoint_count / hard_endpoint_count`
- runtime `last_debug` 也會在 partial / final 被 validator 擋掉時記錄最近原因。

### 測試

- 新增與擴充多組 v2 穩定化、post-processor、UI diagnostics 測試。
- 這批相關測試驗證結果：`187 passed`
