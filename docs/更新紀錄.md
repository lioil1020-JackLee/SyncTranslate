# 更新紀錄

## 2026-04-01

### Runtime Adaptive Tuning
- 新增 `runtime.adaptive_asr_enabled` 與 `runtime.adaptive_llm_enabled`，預設都啟用。
- ASR 會依最近語句長度、partial latency、final latency 與 queue 壓力，動態調整 partial 間隔、VAD silence 與 soft final 切段。
- LLM 會依最近片段長度、翻譯延遲與失敗率，動態調整 partial trigger、最小 partial 間隔、context items，必要時暫時切向 `live_caption_stable`。
- adaptive tuning 只在目前 session 內生效，不會自動把學到的值寫回 `config.yaml`。

### Live ASR 預設
- 將 partial decode 的 `beam_size` 預設改為 `1`，保留 final decode 的 `final_beam_size = 3`。
- 將 `partial_history_seconds` 預設縮短為 `2` 秒，降低 live partial decode 負擔。
- UI 內建 optimized defaults、`config.yaml` 與 `config.example.yaml` 已同步對齊新的 live ASR 預設。

### Diagnostics 與測試
- 主視窗 runtime stats 新增 adaptive mode 與即時生效值顯示，方便觀察 ASR 的 runtime 行為。
- 新增 ASR / LLM adaptive tuning 測試，覆蓋短句、長句、延遲升高與保守模式切換情境。

## 2026-03-30

### 即時字幕與方向控制
- 將即時字幕頁的方向設定整理為三個獨立開關：`ASR語言`、`翻譯目標`、`TTS語音`。
- `ASR語言 = 無` 時會完全停用該方向的辨識鏈，適合會議紀錄或單向翻譯。
- `翻譯目標 = 無` 時只保留原文字幕，不再硬套翻譯流程。
- `TTS語音 = 無` 時只顯示文字，不再被自動補回預設聲線。

### ASR 穩定性
- 調整 ASR 預設為穩定優先，降低 partial / final hallucination。
- 將 passthrough 由同步播放改為非同步佇列，避免 capture callback 被播放裝置阻塞。
- 調整 remote / local 的 ASR 執行順序，改為先提交 ASR，再處理 passthrough。
- 對相同 ASR runtime 組態改回共用單一 Whisper model，並以共享 lock 控制 decoder 存取，避免 GPU 過載。

### 設定與模型
- 設定頁優化預設更新為保守參數組合，降低即時辨識過載。
- `config.yaml` 預設同步更新為較保守的 partial 與 VAD 參數。
- ASR 模型選單移除 `large-v1`，避免誤用舊版模型。

## 2026-03-28

### 即時字幕熱切換
- `即時字幕` 頁面的執行中控件不再因 session 啟動而鎖住。
- `直通 / 翻譯`、翻譯目標、TTS 聲線、ASR 語言現在都可在 session 執行中直接熱切換。
- 熱切換時會同步更新執行中的 runtime 設定，ASR stream 會依新語言設定重建，不需要先停止再開始。

### UI 與互動

- 即時字幕頁新增明確的 `模式` 切換，將 `直通` 與 `翻譯` 從 `翻譯目標` 中拆開。
- 舊版共用 `輸出音量` 已退役，改由 `設定 > 音訊裝置` 的四個真實裝置音量 slider 取代。
- 新版 slider 以 `0% ~ 100%` 顯示真實裝置音量百分比。
- 恢復 `lioil.ico` 的執行期與打包 icon 路徑，避免視窗與 onedir 成品遺失圖示。
- 即時字幕頁移除已停用的 `測試TTS` 按鈕，避免舊 UI 控件再次出現。
- `模式` 已移到 `翻譯目標` 左側。
- 設定頁的快速操作區改成單行摘要，不再被截字。
- 修正 `模式` 切成 `直通` 後會被自動切回 `翻譯` 的問題。

### Runtime 與設定

- 預設參數回調為穩定優先。
- 不再強制使用錯誤的 `live_caption_stable` profile。
- 修正 `config.yaml` 中多組 LLM profile 被錯誤覆蓋的問題。
- `LiveCaptionPage.apply_config()` 現在會優先尊重顯式的 channel mode 設定，不再用目標語言與聲線值反推模式。

### 代碼清理

- 刪除 `LocalAiPage` 中舊版快速設定殘留方法。
- 刪除未使用的 `app/infra/audio/meter.py`。
- 清理 live caption 頁面中未使用的 helper 與 import。
- 將 `LiveCaptionPage` 重構為 channel-centric 寫法，收斂 remote/local 的重複 UI 與設定同步邏輯。
- 補齊進階設定中缺少的 runtime / LLM profile / TTS style 參數，並讓 `config.yaml` 能完整保存對應欄位。

### 文件

- 新增從零開始的 [快速安裝手冊](./快速安裝手冊.md)，包含 LM Studio、模型下載、SyncTranslate 啟動與第一次測試流程。

## 2026-03-26

### 基礎整理

- Runtime 固定採用 `bidirectional`。
- ASR / LLM 保持方向獨立。
- 移除 warmup / preheat 流程。
- 將 health check 統一命名為 system check。
## 2026-03-30

### 真實裝置音量控制

- 移除 `即時字幕` 頁面的共用 `輸出音量`。
- `設定 > 音訊裝置` 新增四個獨立音量控制：`遠端輸入`、`遠端輸出`、`本地輸入`、`本地輸出`。
- 四個 slider 直接控制 Windows 真實裝置的 endpoint volume，包含一般硬體裝置與 VoiceMeeter 虛擬裝置。
- SyncTranslate 內部 capture / playback gain 固定為 `100%`，不再用內部增益模擬真實裝置音量。
## 2026-03-30 TTS 佇列與圖示

- TTS 改成合成與播放分離的雙階段佇列，播放上一句時可先準備下一句，減少句尾等待。
- TTS 直接朗讀 LLM 最終譯文，不再額外產生 speech-profile 版本的 spoken text。
- 預設不再朗讀 stable partial，讓 ASR、LLM、TTS 各自只負責自己的最終輸出。
- `啟用上下文拼接` 已調整為 `僅 partial 使用上下文`，保留短句接話能力，但 final 翻譯不再因前文而漂移。
- 執行中視窗與打包成品持續使用 `lioil.ico` 作為應用程式圖示。
# 2026-03-31

- 四個即時字幕面板的匯出文字檔改成最舊訊息在最上面、最新訊息在最下面。
- 匯出文字檔時會移除 `[final]` 前綴，保留乾淨可讀的字幕內容。
