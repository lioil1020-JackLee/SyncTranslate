## 2026-04-16（Phase 1–4 商品化重構）

### 新增模組

#### Phase 1：ASR 後處理與可觀測性

- `app/application/transcript_postprocessor.py`
  - partial 穩定前綴、text normalization、glossary 套用
- `app/domain/glossary.py`
  - GlossaryStore：支援 substring / exact 模式、case-insensitive 比對
- `app/infra/config/glossary_loader.py`
  - 從 YAML / JSON 載入 GlossaryStore，失敗時回傳空 store
- `app/infra/logging/runtime_logger.py`
  - 背景 worker thread、non-blocking、jsonl 輸出
- `app/domain/metrics.py`
  - UtteranceLatency / PipelineMetrics / MetricsCollector

#### Phase 2：StreamingPolicy 與 ASR Profiles

- `app/infra/asr/streaming_policy.py`
  - StreamingPolicy / StreamingContext / StreamingDecision
  - 三段 degradation：normal / congested / degraded
- `app/infra/asr/endpoint_profiles.py`
  - 6 個內建 endpoint profile：default / meeting_room / headset / noisy_environment / max_accuracy / low_latency

#### Phase 3：音訊前處理鏈與 Benchmark 工具

- `app/infra/asr/audio_pipeline/`
  - AudioFrontendChain、HighpassStage、LoudnessStage、NoiseReductionStage、MusicSuppressionStage
  - 預留 AEC 擴充點（`insert(index, stage)`）
- `tools/asr_benchmark/run_benchmark.py`
  - 離線 ASR benchmark CLI（CER / WER / latency）
- `tools/asr_benchmark/report.py`
  - 報表產生器（table / csv / json）

#### Phase 4：AudioRouter 責任分離 & UI Controller 化

- `app/application/asr_event_processor.py`
  - latency gate → postprocess → transcript 儲存
- `app/application/translation_dispatcher.py`
  - 非同步翻譯 queue、tail-drop overflow 策略
  - 修正：stop() 結束後清空殘留 sentinel，避免下次 start() 瞬間退出
- `app/application/tts_dispatcher.py`
  - 依 channel output mode 決定是否入隊
- `app/application/pipeline_metrics.py`
  - UtteranceMilestones / PipelineMetricsCollector（thread-safe，window_size 可設）
- `app/ui/controllers/session_action_controller.py`
- `app/ui/controllers/live_caption_refresh_controller.py`
- `app/ui/controllers/config_hot_apply_controller.py`
- `app/ui/controllers/healthcheck_controller.py`

## 2026-04-16（FunASR 專屬參數化 + 中文調參回圈）

- `AsrConfig` 新增 `funasr` 專屬設定區塊（含 `use_itn`、offline/online batch、online chunk、短句抑制門檻、benchmark window/overlap）。
- `backend_v2.py` 的 FunASR 路徑改為讀取 `profile.funasr`，並支援 `hotword`、online chunk 參數與可調的短句抑制策略。
- `tools/asr_benchmark/run_benchmark.py` 新增 FunASR window/overlap 參數，並在結果中輸出 FunASR 相關欄位。
- 新增 `tools/asr_benchmark/tune_funasr.py`，可對中文資料集自動 sweep 並輸出最佳參數。
- 以 `downloads/chinese_srt/*.wav` + 對應字幕測試，FunASR 最佳組合為 `recall_bias_1`，`cer_normalized=0.1598`（約 84.02%）。

## 2026-04-16（ASR UI 語意改版：中文 / 非中文）

- Local AI 頁面的 ASR 進階欄位由「本地方向 / 遠端方向」改為「中文 / 非中文」。
- 模型顯示改為固定文字：中文顯示 `FunASR`、非中文顯示 `faster-whisper`（不再提供共用引擎選擇）。
- 新增中文（FunASR）專屬開關：`ITN`、`Online`。
- 執行期 profile 選擇改為語言家族導向：
  - 中文語系 -> 使用中文 profile（FunASR）
  - 非中文語系 -> 使用非中文 profile（faster-whisper）
  - `auto` 維持原通道 profile fallback 以避免舊行為回歸風險
- 中文專攻測試（CUDA）結果：本資料集 FunASR 最佳仍約 `84.02%`，尚未達 95%。
